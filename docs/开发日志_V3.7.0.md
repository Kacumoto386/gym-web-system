# 开发日志 V3.7.0 — 上课记录手机号展示 + 时间校验

**日期**: 2026-05-22  
**类型**: feature, fix

---

## 变更内容

### 1. 列表增加手机号列

上课记录列表新增"手机号"列（会员名和课程之间），数据来源：
- 优先使用 `ClassRecord.member_phone` 已有数据
- 为空时自动从 `Member` 表补充会员手机号（`_build_table` 查询时批量查询 Member 表）

编辑弹窗中手机号改为**只读**（灰色背景 + cursor-not-allowed），标签显示 `(只读)`，手机号和会员姓名只能通过会员管理修改。

### 2. 编辑模态框时间选择器

开始时间和结束时间由普通文本输入改为 `type="time"` 时间选择器，点击弹出系统时间选择面板。

### 3. 结束时间不能早于开始时间

- **前端校验**: `validateEditTime()` — 实时检测，结束时间 ≤ 开始时间时显示红色错误提示，阻止保存
- **后端校验**: `update_class_record()` — PUT 请求时校验 `end_time > start_time`，否则返回 422

### 4. 编辑保存后数据刷新修复

`refreshAll()` 由 `htmx.trigger()` 改为 `fetch()` + `innerHTML`（与 V3.6.7 finance 模块相同模式），确保保存后表格和卡片即时刷新。

### 5. 编辑提交改为 fetch

编辑表单由 `hx-put=""` + `setAttribute` 方式（HTMX 不拾取动态变更的属性）改为 `fetch` + JSON 提交，修复保存按钮 405 错误。
