# 开发日志 V3.6.9 — 到期提醒模块实时数据增强

**日期**: 2026-05-22  
**类型**: feature

---

## 变更内容

### 1. 到期提醒模块重写

到期提醒模块原依赖 `alert` 表，但整个代码库没有任何写入该表的逻辑，导致页面永远显示"暂无提醒"。改为**实时查询源数据表**，按到期类型分组展示。

**数据来源（5种类型）**:

| 类型 | 源表 | 日期字段 |
|------|------|----------|
| 会籍到期 | `MembershipCard` | `end_date` |
| 课程到期 | `LessonPackage` | `valid_until` |
| 包月到期 | `MonthlyPass` | `valid_until` |
| 会员到期 | `Member` | `end_date` |
| 生日提醒 | `Member` | `birth_date` |

每种类型独立查询函数，返回统一格式的 dict 列表，实时计算 `remaining_days`。

### 2. 汇总卡片

页面顶部 4 个卡片：总提醒数、即将到期(≤7天)、已过期、今日生日。各带不同颜色左边框和数值。

### 3. 分组展示

各类型按颜色分区展示（参考 `member_assets.html` 的 sectioned layout）：
- **会籍到期** — 绿色左边框
- **课程到期** — 蓝色左边框
- **包月到期** — 紫色左边框
- **会员到期** — 黄色左边框
- **生日提醒** — 粉色左边框（不同列结构：出生日期 → 即将到来日期 → 天数）

每个分区含标题栏（彩色圆点 + 类型名 + 计数）和子表格。

### 4. 筛选交互

- 类型筛选下拉（全部/会籍到期/课程到期/包月到期/会员到期/生日提醒）
- 状态筛选（全部/即将到期/已过期）
- 搜索框（按会员姓名搜索，支持回车触发）
- 查询/重置按钮

### 5. 状态标签

- 已过期：红色
- 今日到期：橙色
- 即将到期(≤7天)：橙色
- 30天内到期：黄色
- 有效：绿色
