# 开发日志 V3.6.7 — 收入支出报表保存修复

**日期**: 2026-05-22  
**类型**: fix, ui

---

## 变更内容

### 1. 收入支出报表保存按钮修复

"收入支出报表"录入新增收入/支出时无法保存。根因是表单通过 HTMX `hx-post` 以 `application/x-www-form-urlencoded` 格式提交，但后端 Pydantic 模型 `IncomeCreate`/`ExpenseCreate` 需接收 `application/json`。类型不匹配导致 FastAPI 返回 422 Unprocessable Entity，弹窗无法关闭。

**修复方案**：前端改用 `fetch` 将表单数据序列化为 JSON 提交，与后端 Pydantic 模型匹配。保存成功后关闭弹窗并通过 fetch 刷新表格和汇总卡片。

### 2. 数据刷新修复

保存后表格不会立即显示新增数据。原因是 `htmx.trigger()` 重新触发 `load` 事件的方式不可靠。改用原生 `fetch` + `innerHTML` 直接获取并替换表格/汇总内容，确保保存后数据即时刷新。
