## 界面

### 1. 手环管理页面

- **导航入口**：侧边栏"🏷️ 手环管理"
- **工具栏**：[+ 注册手环] [+ 编辑] [🎯 绑定会员] [🔗 解绑] [🗑️ 删除]
- **表格**：手环编号、读卡器写入值、自定义编号、绑定会员编号、绑定会员姓名、绑定状态、绑定时间、注册时间
- **注册对话框**：录入读卡器值(10位数字) + 自定义编号 + 备注
- **绑定会员对话框**：搜索并选择会员 → 确认绑定

### 2. 会员编辑绑手环

- MemberDialog 照片下方新增"🏷️ 手环绑定"区域
- 已绑：显示手环编号+读卡器值+自定义编号 + [🔗 解绑] 按钮
- 未绑：显示"未绑定手环" + [🎯 绑定手环] 按钮
- 点击绑定 → BandSelectDialog 弹出未绑定手环列表 → 选一个绑定

### 3. 刷卡进场

- CheckinPanel 查询区下方新增"刷手环进场"区域
- [🏷️ 刷卡] 按钮 → 切换刷卡模式(激活手环输入框)
- 读卡器模拟键盘输入10位数字+回车 → `_on_band_search()`
- 查找流程：find_by_reader_value() → 检查绑定 → get_member() → 填充到核销面板
- 未识别：提示"未识别的手环"
- 未绑定：提示"手环未绑定会员"

## 关键决策

1. **手环编号规则**：WB+年月日+4位序号（与现有编号风格一致）
2. **读卡器值唯一性**：注册时校验不可重复
3. **绑定关系**：1:1，简单清晰
4. **刷卡模式**：切换式，不干扰现有编号查询流程
5. **读卡器兼容**：回车触发查询 = 兼容模拟键盘输入的RFID/IC读卡器

## V2.16.2 Bug 修复

**修复时间**: 2026-05-08 17:57

### Bug 1: 手环注册无法保存
- **现象**: 注册手环时数据无法持久化到 Excel
- **根因一**: _init_wristband_sheet() 只检查 Sheet 是否存在，但手环管理 Sheet 已存在却无表头数据（1行1列空单元格），导致 get_header_col() 全部返回 None，数据写入空行
- **根因二**: dd_wristband() 中 self.engine.append_row(sheet_name) 只传1个参数，但 ppend_row(self, sheet_name, data_dict) 需要2个参数，会抛 TypeError
- **修复**: 
  1. _init_wristband_sheet() 新增 Sheet 存在但无表头时的初始化分支（写入9列表头）
  2. ppend_row → ind_next_empty_row 正确获取行号
