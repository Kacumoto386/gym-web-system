# V2.15.8 教练排班日历化改造方案

## 版本信息
- **版本**: v2.15.8
- **日期**: 2026-05-08
- **类型**: 功能重构 + UI优化 + Bug修复
- **前序版本**: v2.15.7 (进场核销UI优化)

## 改动背景
现有排班页面为「表格视图」（行=时间段，列=教练），信息密度低且不直观。
教练日常需要看的是"某天有哪些课"，而非"某时间段有哪些预约"。
改为月历形式更符合日常使用习惯。

### 月历格子
- 紧凑型设计，每个格子显示日期数字 + 课节数
- 状态底色：浅绿(已预约) / 浅橙(已签到) / 浅蓝(已完成)
- 选中日期高亮(浅黄)
- 今天标记圆点

### 右侧统计
- 月度总课节、有课天数
- 待签到、已签到、已完成数量（不含已取消）

### 下方详情
- 点击日期展开，Treeview 列表显示当日全部预约
- 状态统计摘要行

## Bug修复
1. **教练下拉为空** — `get_all_trainers()` 只查"职务"字段，Excel 中是"岗位"
2. **删除上课记录报错** — 传入多余参数 `class_id`
3. **详情区残留边框** — `_clear_detail` 只销毁 Treeview 未清理 Scrollbar
4. **UI美观** — 月历紧凑化、左右等宽双栏、统计数据卡片

## 数据流
1. 选择教练 + 月份 → 调用 `biz.get_monthly_schedule(coach, year, month)`
2. 后端汇总该教练当月每天的预约数据
3. 按日期填充日历格子，点击日期显示当日预约详情

## 兼容性
- 无破坏性变更
- 数据字段完全沿用现有预约系统
