# 开发日志 V3.2.9 — 系统名称自定义功能

**日期**: 2026-05-13
**类型**: feature

---

## 需求

在操作日志模块增加一个小功能，可以修改当前系统名称。

## 方案设计

### 存储方式
使用 `system_settings` 表（KV 存储），key=`system_name`，value=用户自定义名称。默认值为"健身房管理系统"。

### API 设计
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/system/settings` | 获取所有系统设置 |
| PUT | `/api/system/settings` | 更新系统设置（body: `{name: "xxx"}`） |

PUT 自动记录操作日志：
- 资源: "系统设置"
- 详情: "修改系统名称: 「旧名称」→「新名称」"

### 前端设计
在操作日志页面顶部新增系统名称编辑区：
- 输入框（最大 50 字符）
- 保存按钮（蓝色）
- 保存状态提示（绿色"已保存" / 红色错误）
- 页面加载时自动读取当前系统名称填入输入框

侧边栏标题同时更新：
- Jinja2 模板全局函数 `_get_system_name()` 每次渲染时从 DB 读取
- JS 页面加载时 fetch `/api/system/settings` 二次刷新（保存后即时生效）

## 实现过程

### 数据库层
- 在 `operation_log.py` 中新增 `SystemSetting` ORM 模型（id/key/value/updated_at）
- `create(bind=engine, checkfirst=True)` 自动建表（已存在则跳过）

### 后端
- `get_system_name(db)` — 读取，不存在返回默认"健身房管理系统"
- `set_system_name(db, name)` — 写入（upsert 逻辑）
- `class SystemNameUpdate(BaseModel)` — Pydantic 输入模型（name: str）
- `system_router` — GET/PUT 路由，PUT 自动调用 `record_log()`

### 前端
- `logs.html` — 顶部编辑区（⚙️ 系统名称 [输入框] [保存]）
- `base.html` — 标题改为 `{{ _get_system_name() }}` + JS 二次刷新

### 版本号更新
- app.py 版本号: 3.2.8 → 3.2.9
- health API 返回 3.2.9
