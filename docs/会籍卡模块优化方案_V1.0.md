# 会籍卡模块优化方案 V1.0

## 1. 需求概述

### 1.1 当前状态
会籍卡模块基本功能已实现（后端 CRUD + 前端页面），包含：
- 卡产品管理（次卡/期限卡/现金卡 三种类型）
- 售卡功能（选择会员+产品+日期）
- 已售记录查看

### 1.2 本轮优化目标

| # | 需求 | 说明 |
|---|------|------|
| 1 | **卡产品新增/编辑位置优化** | 界面更清晰，字段分组合理 |
| 2 | **保留选择日期 + 到日期展示** | 售卡时日期选择和自动计算截止日期已有，需确保体验流畅 |
| 3 | **现金卡面值+售价** | 面值（仅展示不关联计算）+ 售价 = 实际储值金额 |
| 4 | **次卡赠送次数** | 新增产品时可选赠送次数 + 已售记录展示赠送次数 |

---

## 2. 数据模型

### 2.1 关联字段说明（已存在）

| 字段 | 类型 | 所属卡类型 | 含义 |
|------|------|------------|------|
| `total_classes` | int | 次卡 | 基础可用次数 |
| `bonus_classes` | int | 次卡 | 赠送次数（仅展示，不影响核心扣减） |
| `face_value` | float | 现金卡 | 面值（仅展示，如 ¥1000） |
| `price` | float | 全部 | 售价 / 储值金额（实际金额） |
| `duration_days` | int | 期限卡 | 有效期（天） |
| `start_date` | date | 售卡实例 | 开卡日期 |
| `end_date` | date | 售卡实例 | 截止日期（自动计算） |

### 2.2 字段关系

```
现金卡: face_value (展示面值) ≠ price (实际储值)
次卡:   total_classes + bonus_classes (赠送) = 可用总次数
期限卡: start_date + duration_days → end_date (自动)
```

## 4. 交互逻辑

### 4.1 卡类型联动

```
选择"次卡"  → 显示: 可用次数 + 赠送次数
选择"现金卡" → 显示: 面值 + 售价(=储值金额)
选择"期限卡" → 显示: 有效期(天)
```

### 4.2 日期自动计算

```
选开卡日期 + 有效期(天) → 自动算到日期
开卡日期为空 → 默认今天
有效期=0 或空 → 不计算到日期（不限期）
```

### 4.3 售卡记录展示

- 次卡售卡：已售表格「次数/面值」列显示 `10次(赠5)`
- 现金卡售卡：已售表格「次数/面值」列显示 `面值¥1000`
- 期限卡售卡：已售表格「次数/面值」列显示 `-`

### 5.3 具体改动说明

#### 5.3.1 新增产品对话框字段分组优化

当前表单采用统一显示全部字段的方式。改为**按卡类型动态显示/隐藏分组**：
- 选择"次卡"时：显示「可用次数」+「赠送次数」行
- 选择"现金卡"时：显示「面值」行
- 通用字段（名称、有效期、售价）始终显示
- 起止日期作为可选字段保留

#### 5.3.2 售卡对话框信息展示优化

当前已有面值和赠送展示。优化点：
- 产品信息区增加更清晰的布局展示
- 输入实收金额（现金卡默认带出产品定价，可修改）

#### 5.3.3 已售表格赠送/面值列

已通过 `_sold_content()` 实现，但表格表头需清晰标注「次数/面值」列
