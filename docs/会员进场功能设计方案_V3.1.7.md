# 会员进场功能设计方案 V3.1.7

**版本**: v3.1.7  
**日期**: 2026-05-09  
**项目**: gym-web-system  
**定位**: 完善现有进场核销页面的缺失能力

---

## 1. 现状分析

现有 checkin.html 已有基础框架：
- ✅ 手动搜索会员 + 扫码/刷卡模式切换
- ✅ 会员选择弹窗
- ✅ 会籍卡片选择
- ✅ 核销方式单选（次卡扣次/储值扣款/期限卡签到）
- ✅ 扣减预览
- ⚠️ 后端扣减已实现（checkin.py），但缺少自动计算逻辑
- ❌ **缺失核心能力**：条形码/二维码扫码进场、自动查会员会籍卡、核销方式引导选择

### 设计的三大缺失细化

| 需求 | 当前状态 | 目标 |
|------|---------|------|
| **扫码进场** | 仅支持手动输入手机号/会员编号 | 支持条形码/二维码扫码自动识别会员（扫码枪模拟键盘输入+回车） |
| **自动查会员会籍卡** | 需要手动搜索会员后才能查看会籍卡 | 扫码即自动查询会员信息+有效会籍卡列表+自动推荐最优核销方式 |
| **核销方式引导选择** | 用户自行选择，无智能推荐 | 根据会籍卡类型+课时/余额智能推荐核销方式，一键确认 |

## 2. 方案设计

### 2.1 功能流程图

```
[扫码枪输入] ──→ 会员编号/手机号/卡号 (回车)
        │
        ▼
  查询会员信息
        │
        ├── 未找到 ──→ 提示"未识别会员，请手动搜索"
        │
        ▼ 找到会员
  查询有效会籍卡
        │
        ├── 无有效会籍卡 ──→ 提示"无有效会籍卡，请先购卡"
        │
        ▼ 有有效会籍卡
  智能推荐核销方式
        │
        ├── 次卡 → 推荐"次卡扣次"（有剩余课时）
        ├── 储值 → 推荐"储值扣款"（有余额）
        ├── 期限卡 → 推荐"期限卡签到"（有效期内）
        └── 多卡 → 推荐优先级：期限卡 > 次卡 > 储值
        │
        ▼
  扣减预览（实时展示）
        │
        ▼
  确认进场 → 记录进场 + 扣减资源
```

### 2.2 扫码进场（核心）

**原理**：条形码/二维码扫码枪 = 键盘模拟器。扫码后自动输入会员编号/手机号 + 模拟回车。

**支持三种码格式**：
1. **会员编号**：如 `M20260508XXXX` → 直接查会员
2. **手机号**：11位数字 → 按手机号查会员
3. **手环读卡器值**：10位数字 → 通过手环绑定查会员

**扫码输入增强**：
- 任何模式下，全局监听快速输入
- 连续快速输入（<50ms间隔）+ 回车 → 视为扫码
- 支持条形码（Code128/Code39/ENA13）和二维码（QR）

### 2.3 自动查会员 + 会籍卡联动

**扫码/输入后自动执行**：
```
1. 识别输入类型 → 会员编号/手机号/手环值
2. POST /api/checkin/quick-lookup 查询
3. 返回：会员信息 + 有效会籍卡列表 + 课程包课时 + 储值余额
4. 自动填充UI：会员信息 → 会籍卡选择 → 推荐核销方式
```

**新增后端端点**：

```
GET /api/checkin/quick-lookup?q={input}
```

返回格式：

5. 用户只需点击"确认进场"

### 2.4 核销方式智能推荐

**推荐优先级逻辑**：

```
1. 检查会员状态 → 非"正常"不推荐
2. 检查有效会籍卡 → 按优先级排序
3. 对每张有效卡推荐最优核销方式：

   次卡类型:
     - 有剩余课时 → 推荐"次卡扣次"
     - 无剩余课时 → 跳过，推荐下一张卡

   期限卡类型（月卡/季卡/年卡/时卡）:
     - 在有效期内 → 推荐"期限卡签到"
     - 已过期 → 跳过

   储值余额:
     - 余额 ≥ 单次进场费(¥30) → 推荐"储值扣款"
     - 0 < 余额 < 30 → 推荐"储值扣款"（扣全部）
     - 余额 = 0 → 不推荐

4. 最终推荐：一张卡一个最优核销方式
5. 用户可手动切换
```

**前端推荐展示**：
- 核销方式区域顶部显示推荐标签 `⭐ 推荐`
- 自动选中推荐项，按钮高亮
- 扣减预览展示"本次扣减"详情

### 2.5 核销方式选择（三种）

| 核销方式 | 适用卡类型 | 扣减逻辑 | 后端实现 |
|---------|-----------|---------|---------|
| **次卡扣次** | 次卡 | `remaining_lessons -= 1` `used_lessons += 1` | ✅ 已有 |
| **储值扣款** | 现金卡/任意有余额 | `balance -= min(balance, 30.0)` | ✅ 已有，可配置单次进场费 |
| **期限卡签到** | 月卡/季卡/年卡/时卡 | 仅签到，不扣减 | ✅ 已有 |

**增强点**：
- 储值扣款增加 "自定义扣款金额" 选项（默认¥30，可手动修改）
- 次卡扣次支持 "自定义扣减次数"（可扣1~5次）
- 期限卡签到显示 "剩余有效天数"

### 2.6 UI 改造方案

**进场页面现有布局不变，增强以下点**：

## 4. API 设计

### 新增：快速进场查询

```
GET /api/checkin/quick-lookup?q={query}
```

支持 query 类型：
- 会员编号（M开头）
- 手机号（11位纯数字）
- 手环读卡器值（10位纯数字，通过手环表关联）
- 自定义码

返回：
```json
{
  "found": true,
  "method": "member_id|phone|wristband",
  "member": {...},
  "active_cards": [...],
  "recommended_consume": {
    "card_id": "MC...",
    "consume_type": "次卡扣次|储值扣款|期限卡签到",
    "label": "推荐说明",
    "fee": 30.0,
    "quantity": 1
  }
}
```

### 增强：进场POST增加参数

现有 `POST /api/checkins` 增加参数：
- `quantity: int = 1` — 扣减数量（次卡扣多次）
- `custom_fee: float = 30.0` — 自定义扣款金额
- `auto_lookup: bool = False` — 标识是否为自动进场（用于日志）

## 5. 数据流

```
用户扫码
    │
    ▼
输入框捕获 (oninput + onkeydown)
    │ 连续快速输入(毫秒级) + Enter
    ▼
fetch /api/checkin/quick-lookup?q=...
    │
    ▼
后端处理:
    1. 识别输入类型
    2. 查 Member 表
    3. 查 MembershipCard 表（status=正常, end_date>=today）
    4. 查 Wristband 表（如果是10位数字）
    5. 推荐算法排序
    6. 返回 JSON
    │
    ▼
前端自动填充:
    1. 会员信息卡片（名称、等级、剩余课时、余额）
    2. 会籍卡下拉（自动选最优卡）
    3. 核销方式自动选中推荐项
    4. 扣减预览刷新
    │
    ▼
用户确认 → POST /api/checkins → 后台扣减 → 刷新表格 → 清空准备下一次
```

## 6. 推荐算法详情

```python
def recommend_consume(member, active_cards):
    """
    智能推荐最优核销方式
    返回排序后的推荐列表
    """
    recommendations = []
    
    for card in active_cards:  # 已按优先级排序
        card_type = card.card_type
        
        # 期限卡：有效期内推荐签到
        if card_type in ('月卡', '季卡', '年卡', '时卡'):
            if card.end_date and card.end_date >= date.today():
                days_left = (card.end_date - date.today()).days
                recommendations.append({
                    "card_id": card.card_id,
                    "card_type": card_type,
                    "consume_type": "期限卡签到",
                    "label": f"{card_type}有效，剩余{days_left}天",
                    "score": 100,  # 最高优先级
                    "fee": 0,
                    "quantity": 1,
                })
        
        # 次卡：有剩余课时推荐扣次
        if '次' in card_type:
            if member.remaining_lessons > 0:
                recommendations.append({
                    "card_id": card.card_id,
                    "card_type": card_type,
                    "consume_type": "次卡扣次",
                    "label": f"次卡，剩余{member.remaining_lessons}课时",
                    "score": 80,
                    "fee": 0,
                    "quantity": 1,
                })
    
    # 储值扣款作为最后兜底
    if member.balance > 0:
        fee = min(member.balance, SINGLE_ENTRY_FEE)
        recommendations.append({
            "card_id": "",
            "card_type": "储值",
            "consume_type": "储值扣款",
            "label": f"余额¥{member.balance:.0f}，扣¥{fee:.0f}",
            "score": 60,
            "fee": fee,
            "quantity": 1,
        })
    
    # 按 score 降序排列
    recommendations.sort(key=lambda r: r['score'], reverse=True)
    return recommendations
```

## 7. UI 交互细节

### 扫码输入检测

```javascript
// 扫码枪检测逻辑
let scanBuffer = '';
let scanTimer = null;
const SCAN_INTERVAL = 50;  // 50ms内连续输入视为扫码

function onScanInput(key) {
    if (key === 'Enter') {
        // 扫码完成
        executeQuickLookup(scanBuffer);
        scanBuffer = '';
    } else {
        scanBuffer += key;
        clearTimeout(scanTimer);
        scanTimer = setTimeout(() => {
            // 超时，可能是手动输入
            scanBuffer = '';
        }, SCAN_INTERVAL * 3);
    }
}
```

## 8. 异常处理

| 场景 | 处理方式 |
|------|---------|
| 扫码未识别 | 提示"未识别此码，请手动搜索会员" + 自动切换到搜索模式 |
| 会员已过期 | 提示"会员已过期，请续费" + 提供续费链接 |
| 所有会籍卡已过期 | 提示"会籍卡已全部过期，请购买新卡" |
| 次卡无剩余课时 | 提示"次卡已耗尽，建议购卡或选择其他方式" |
| 储值余额不足 | 提示"余额¥X不足进场费¥30，请充值" |
| 网络错误 | 提示"网络故障，请重试" + 自动重试1次 |
| 重复进场（同会员同一天多次） | 提示"该会员今日已进场N次"（不阻止，仅提示） |
