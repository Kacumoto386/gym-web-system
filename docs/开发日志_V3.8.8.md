# 开发日志 V3.8.8 — 功能配置清单 + Bug 修复

**日期**: 2026-05-26  
**类型**: feat, fix

---

## 变更内容

### 1. 功能配置清单（Feature Flag）系统

**背景**: 各健身房业务需求不同，需要一套配置体系，让客户通过编辑配置文件就能启用/禁用功能模块。

**改动**:
1. **`backend/features.yaml`（新增）** — 客户可编辑的 27 项功能开关配置文件，按 6 大分组（会员/课程/进场/员工/财务/系统）组织
2. **`backend/feature_registry.py`（新增）** — 功能注册表，定义全部功能元数据（FEATURE_DEFS + NAV_TREE），加载 YAML 配置，提供查询接口
3. **`backend/app.py`（修改）** — 条件注册路由 + 模板全局变量注入 `_nav_tree` / `_group_names`
4. **`frontend/templates/base.html`（修改）** — 导航菜单从硬编码改为 Jinja2 动态渲染 + `window._GROUP_NAMES` 注入
5. **`frontend/static/js/app.js`（修改）** — GROUP_NAMES 从模板注入读取
6. **`requirements.txt`（修改）** — 添加 `PyYAML>=6.0`

**设计要点**:
- 客户只需编辑 `features.yaml`，不需要改任何 Python 代码
- 设为 `false` 后：导航隐藏 + 页面路由 404 + API 路由不注册
- 配置文件缺失/错误时自动启用所有功能（零成本回退）

### 2. 会员录入 JSON 显示问题修复

**背景**: 新增会员后页面直接显示 JSON 文本，刷新后才正常渲染。

**根因**: `create_member` 接口 `response_model=MemberOut` 返回 JSON，而 HTMX 表单期望 HTML 响应。

**修复**: 
- 移除 `response_model=MemberOut`，改用 `HX-Redirect` 响应头让 HTMX 做完整页面跳转
- 避免 303 重定向导致的 JavaScript 重复加载报错（`const` 重复声明）

### 3. 会员身高/体重/体脂默认值修复

**背景**: 未填身高/体重/体脂的新会员默认显示 0/0.0。

**根因**: Pydantic schema 中 `height`、`weight`、`body_fat` 默认值为 `0` 而非 `None`。

**修复**: 默认值改为 `None`，未填时 JSON 输出 `null`。

### 4. 前台快速签到剩余次数显示错误

**背景**: 会员有次卡（20次+5赠送）但前台签到显示剩余 0 次。

**根因**: UI 售卡 `sell_card_to_member()` 创建 MembershipCard 记录后未更新 `Member.remaining_lessons`。导入流程有此逻辑但售卡流程遗漏。

**修复**:
- `sell_card_to_member()` 售出次卡后自动累加 `total_classes + bonus_classes` 到 `Member.remaining_lessons`
- 执行一次数据修复，补回已有会员的 `remaining_lessons`
