# 开发日志 V3.8.9 — 看板图表修复 + HTMX 删除刷新 + 小程序预开发

**日期**: 2026-05-27  
**类型**: fix, feat

---

## 变更内容

### 1. 数据分析看板显示修复

**背景**: 数据分析看板页面图表容器无限拉长，控制台报错。

**根因**: 图表卡片没有固定高度，Chart.js 无法正确计算 canvas 尺寸。

**修复**: 
- 所有 8 张图表卡片添加 `style="height:280px"` 固定高度
- canvas 外层包裹 `div style="height:220px"`，canvas 使用 `class="w-full h-full"`
- 空数据时在 canvas 上显示"暂无数据"文字提示（而非空白）

### 2. 首页趋势图数据修复

**背景**: 首页"趋势图"Tab 中注册趋势和营收趋势显示空白/零值。

**根因**:
- 注册趋势：`func.date()` 返回字符串，但 `reg_map.get(d, 0)` 使用 `date` 对象作为 key，类型不匹配导致查不到数据
- 月度营收：仅从 `Sale` 表查询，遗漏了 `FinanceIncome` 表的收入数据

**修复**:
- 注册趋势：`reg_map.get(d, 0)` → `reg_map.get(d.isoformat(), 0)`
- 月度营收：新增 `FinanceIncome` 数据源，跨 `Sale` + `FinanceIncome` 两张表汇总

### 3. 进场高峰时段类型修复

**背景**: 数据分析看板的进场高峰时段图表数据异常。

**根因**: `func.substr()` 在 SQLite 中返回字符串类型，`hour_map` 的 key 是字符串，但遍历 `range(6, 23)` 生成整数 key → `hour_map.get(h, 0)` 查不到。

**修复**: `func.substr(...)` → `cast(func.substr(...), Integer)`，确保 hour_map 的 key 类型一致。

### 4. HTMX 删除操作显示 JSON 修复

**背景**: 多个页面（课程管理、体测记录等）点击删除后显示裸 JSON `{"success":true,"message":"..."}`，需手动刷新。

**根因**: `hx-delete` 按钮发送 DELETE 请求，后端 DELETE 端点返回 Python 字典 → FastAPI 自动序列化为 JSON，HTMX 收到 JSON 后直接替换目标元素内容。

**修复**:
- 新增 `hx_refresh_middleware`：拦截所有 DELETE 请求，成功（2xx）且响应为 JSON 时自动添加 `HX-Refresh: true` 响应头
- HTMX 检测到该头后执行完整页面刷新，解决所有 DELETE 操作的 JSON 显示问题

**额外修复**: `app.py` 中 `JSONResponse` 在异常处理器中使用但未导入的遗留 Bug（触发 404/500 时代码崩溃）。

### 5. 会员小程序预开发

**背景**: 为系统扩展微信小程序端，完成基础框架搭建。

**改动**:
1. **数据库模型** — 新增 `MemberWechatAccount`（会员微信映射）、`StaffWechatAccount`（员工微信映射）、`Package`（可售课程包/商品）
2. **小程序路由注册** — `app.py` 注册 `member_miniapp_router`，受 `features.yaml` 控制
3. **功能开关** — `features.yaml` 新增 `miniapp_staff` / `miniapp_member` 开关
4. **中间件白名单** — `PUBLIC_API_PREFIXES` 新增 `/api/miniapp/staff/`、`/api/miniapp/member/`
5. **异常处理器** — 注册 `MiniAppException` 全局异常处理器
