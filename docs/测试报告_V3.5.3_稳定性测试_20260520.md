## 发现并修复的 BUG（6 个）

### BUG-1: finance.py — `response_model=List` 导致 500
- **严重度**: 高
- **文件**: `backend/routers/finance.py:181,195`
- **现象**: `GET /api/finance/income` 和 `GET /api/finance/expense` 返回 500
- **根因**: `response_model=List`（裸泛型，未指定元素类型），Pydantic v2 无法序列化 ORM 对象
- **修复**: 移除 `response_model=List`，让 FastAPI 自动使用 `jsonable_encoder` 处理

### BUG-2: membership_card.py — 同上
- **严重度**: 高
- **文件**: `backend/routers/membership_card.py:397`
- **现象**: `GET /api/membership-cards` 返回 500
- **根因**: 同 BUG-1
- **修复**: 同上

### BUG-3: body_measurement.py — `measure_date` None 校验失败
- **严重度**: 高
- **文件**: `backend/routers/body_measurement.py:82`
- **现象**: `GET /api/body-measurements` 返回 500
- **根因**: `BodyMeasurementOut.measure_date: date` 为必填，但数据库存在 NULL 值行
- **修复**: 改为 `measure_date: Optional[date] = None`

### BUG-4: commission.py — 缺少 GET /tiers 端点
- **严重度**: 中
- **文件**: `backend/routers/commission.py`
- **现象**: `GET /api/commission/tiers` 返回 405 Method Not Allowed
- **根因**: 仅有 `/tiers/table`(HTML) 和 `/tiers/list`(JSON)，缺少 `/tiers` 直接访问
- **修复**: 新增 `@router.get("/tiers")` 端点返回 JSON 列表

### BUG-5: chat_router.py — 无效 Python 转义序列
- **严重度**: 低
- **文件**: `backend/routers/chat_router.py:423-424`
- **现象**: 启动时 Python 输出 `SyntaxWarning: invalid escape sequence '\/'`
- **根因**: 嵌入 HTML 中的 JS 正则含 `<\/`，Python 将 `\/` 视为无效转义序列
- **修复**: 将字符串内 `\/` 转义为 `\\/`

### BUG-6: test_stability.py — form/json 请求类型混用
- **严重度**: 低（测试代码 BUG，非系统 BUG）
- **文件**: `test_stability.py`
- **现象**: staff/course 创建测试发 form data 而端点期望 JSON body (Pydantic 模型)
- **修复**: `data=` 改为 `json=`
