## 梯度提成引擎

### 核心方法（core/commission_mixin.py）

| 方法 | 说明 |
|:-----|:------|
| `get_tier_rules(type, only_enabled)` | 读取梯度规则，按排序号升序 |
| `add_tier_rule(data)` | 新增梯度规则（自动生成 T+日期+4位编号） |
| `update_tier_rule(row, data)` | 编辑梯度规则 |
| `delete_tier_rule(row)` | 删除梯度规则 |
| `match_tier_rate(type, value)` | 全额匹配：找到值所在档位，返回(梯度名称, 提成率, 下限, 上限) |
| `calculate_tier_commission(type, value)` | **分段累进**计算：返回 `{total_commission, rate, segments[]}` |
| `calc_sale_commission_by_tier(name, amount)` | 员工售课梯度提成（兼容固定比例） |
| `calc_class_commission_by_tier(name, count)` | 员工上课梯度提成（兼容固定比例） |

### 两种计算模式

#### ① 全额匹配（match_tier_rate）
直接找到值所在的档位，**全部销售额**按该档提成率计算。
```
月销售额 50000 → 匹配到 3~8万档（8%）→ 提成 = 50000 × 8% = 4000
```

#### ② 分段累进（calculate_tier_commission）⭐ 推荐
将销售额拆分到各档区间，各区间分别计算再累加。
```
月销售额 120000：
  0~3万     × 5%  = 1500
  3~8万     × 8%  = 4000
  8~12万    × 10% = 4000
  ─────────────────
  总计 9500（综合提成率 7.9%）
```

### 兼容旧逻辑
- 员工表中仍保留「售课提成比例」「上课提成比例」字段
- `calc_sale_commission_by_tier()` 和 `calc_class_commission_by_tier()` 优先使用梯度规则
- 如果无梯度规则或梯度规则返回空，回退到固定比例计算

## 默认梯度规则

### 售课提成（按销售额）

| 档位 | 下限 | 上限 | 提成率 |
|:-----|:----:|:----:|:------:|
| 初级售课 | 0 | 3万 | 5% |
| 中级售课 | 3万 | 8万 | 8% |
| 高级售课 | 8万 | 15万 | 10% |
| 顶级售课 | 15万 | ∞ | 12% |

### 上课提成（按上课节数）

| 档位 | 下限 | 上限 | 提成(元/节) |
|:-----|:----:|:----:|:----------:|
| 初级上课 | 0 | 30节 | 5 |
| 中级上课 | 30节 | 60节 | 8 |
| 高级上课 | 60节 | ∞ | 10 |

> 上课提成按节数分段累进 × 100元/节的基准价估算

## 界面改造（gui/stats_frame.py）

### CommissionFrame 升级内容
1. **统计期间筛选**: 本月 / 上月 / 本年 / 全部
2. **梯度配置入口**: 点击"⚙️ 梯度配置"打开独立管理界面
3. **详细/简洁模式**: 点击"📋 显示详细分段"可展开分段明细
4. **梯度状态提示**: 显示当前售课/上课各有几档规则
5. **表格字段增强**: 售课金额/上课节数 → 分段累进计算，展示梯度名称

### 梯度配置界面（gui/commission_tier_frame.py）
- `CommissionTierFrame`: 主表格 + 类型切换 + 预览计算 + 增删改按钮
- `CommissionTierDialog`: 新增/编辑对话框（8个字段，含校验）
- 预览功能：输入金额点击"🔍 试算"，显示分段明细和最终提成
