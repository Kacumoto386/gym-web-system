# 开发日志 V3.5.9 — 预约管理增加课程购买校验 + 会员搜索改进

**日期**: 2026-05-22
**类型**: feature + bugfix
**触发**: 用户反馈会员可预约所有课程无购买校验；售课记录中会员搜索不到已有会员

---

## 背景

预约管理（Booking）创建预约时，会员可以预约任意课程，没有校验是否已购买。同时 LessonPackage 表已记录了每个会员的已购课程和剩余课时，但未被利用。

售课记录新增售课时，会员搜索只加载了最新 10 个会员到本地缓存做前端过滤，导致大部分会员搜索不到。

---

## V3.5.9 — 预约购买校验

### 修改详情

#### 后端：`backend/routers/booking.py`

| 变更 | 说明 |
|------|------|
| **新增** `GET /api/booking/member-courses?member_id=X` | 查询 LessonPackage 表，按课程分组返回该会员的已购课程、剩余课时、是否过期 |
| **修改** `POST /api/booking/create` | 在教练冲突检测后增加 LessonPackage 校验，无有效课程包时返回 400 |
| **删除** `GET /api/booking/check-availability` | 不再需要单独的可用性检查端点（已合并到 member-courses） |

**校验规则**：LessonPackage 满足 `status="正常"` AND `remaining_hours>0` AND（`valid_until IS NULL` OR `valid_until>=today`），且课程匹配（`course_id` 直接匹配或 `included_courses` 包含）。

#### 前端：`frontend/templates/booking.html`

| 变更 | 说明 |
|------|------|
| **新增** 会员搜索框 | 替换原来的 `<select>` 下拉，支持输入手机号/姓名即时搜索并选中 |
| **新增** `loadCoursesForMember()` | 选中会员后自动加载该会员的已购课程到课程下拉 |
| **删除** `loadCourses()` | 不再加载全量课程列表（改为 member-courses 端点点按需加载） |
| **删除** `checkAvailability()` / `_bookingAvailable` | 不再需要单独的可用了检查 |

**课程下拉展示**：
- 有效课程：`私教课 (剩余 10 课时)`
- 已过期课程：`团课 (已过期)`
- 无课程：`该会员暂无已购课程`

## V3.5.9 — 会员搜索修复

### 问题

售课记录新增售课时，`loadMembers()` 调用 `/api/members/search-json?q=` 只返回最新 **10 个**会员到 `membersCache`，随后 `filterMembers()` 仅在这个 10 条缓存中过滤。若目标会员不在最新的 10 条中，则搜索不到。

### 修复

| 文件 | 变更 |
|------|------|
| `frontend/templates/sales.html` | `filterMembers()` 改为每次输入调 API（200ms 防抖），`showMemberList()` 统一渲染结果 |
| `backend/routers/member.py` | `search-json` 端点返回上限 10 → **50** |

## 验证方法

1. 预约管理 → 新增预约 → 输入会员手机号搜索 → 选中后课程下拉只显示已购课程
2. 选择无购买记录的会员 → 课程下拉显示"该会员暂无已购课程"
3. 选择已过期的课程包 → 课程名后显示"已过期"
4. 通过 API 直接提交无效预约 → 返回 400
5. 售课记录 → 新增售课 → 搜索会员 → 能搜到数据库中任意会员
