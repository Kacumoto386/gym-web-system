# 业绩统计模块改造方案 V2.15.1

## 一、改造目标

### 现状
- 模块名称：**统计分析**
- 包含3个子页面：售课统计、上课统计、员工佣金
- 售课统计只展示课程销售数据（SaleStatsFrame）
- 缺少：**课程包业绩**、**会籍卡业绩**、**会员进场记录**

### 目标
- **模块重命名**：统计分析 → **业绩统计**
- **保留并增强**：售课统计（增加到期时间展示）、上课统计、员工佣金
- **新增**：课程包业绩统计、会籍卡业绩统计、会员进场统计

---

## 二、数据源分析

| 业绩维度 | 数据源Sheet | 核心字段 |
|---------|------------|---------|
| 售课 | 售课记录 | 售课金额、实收金额、售课日期、有效期截止日、支付方式 |
| 课程包 | 课程包管理 | 总课时、已消耗课时、剩余课时、有效期起止、价格 |
| 会籍卡 | 会籍卡管理 | 卡类型(次卡/期限/现金)、售价、实收金额、出售日期、有效期 |
| 会员进场 | 进场记录 | 进场时间、会员姓名、进场方式(签到/刷卡)、关联卡号 |

- **新增"到期售课"高亮提示**

#### ② 售课业绩（原售课统计，增强）`gui/stats_frame.py -> gui/performance_sale_frame.py`
- **保留**原有售课统计功能
- **新增**到期时间展示列：有效期截止日、剩余天数
- **新增**到期状态标签：正常/即将到期(<7天)/已过期

- 统计卡片新增"已到期售课金额""即将到期售课金额"

#### ③ 课程包业绩（新增）`gui/performance_package_frame.py`
- 数据来源：课程包管理Sheet（`lesson_package`）
- 统计维度：
  - 已售课程包总数（按时间筛选）
  - 课程包总金额
  - 已消耗/剩余课时统计
  - 按课程类型统计
- 卡片指标：
  - 已售包数、总金额、平均单价
  - 已消耗课时/总课时
  - 活跃包数（有剩余课时）
- 详细表格：课程包编号、会员姓名、总课时、已用、剩余、状态、有效期
- 时间筛选：售出日期筛选

#### ④ 会籍卡业绩（新增）`gui/performance_membership_frame.py`
- 数据来源：会籍卡管理Sheet（`membership`）
- 统计维度：
  - 按卡类型（次卡/期限卡/现金卡）分类统计
  - 售卡数量、总金额
  - 有效期分布
- 卡片指标：
  - 总售卡数、总金额、平均单价
  - 次卡销售额、期限卡销售额、现金卡销售额
  - 有效卡数/已过期卡数
- 详细表格：会籍卡编号、会员姓名、卡类型、售价、有效期、状态
- 时间筛选：售出日期

#### ⑤ 会员进场（新增）`gui/performance_checkin_frame.py`
- 数据来源：进场记录Sheet（`checkin`）  
- 统计维度：
  - 进场人次统计（今日/本周/本月）
  - 按进场方式统计（次卡/现金卡/期限卡/免费入场）
  - 时段分布（早晨/上午/下午/晚上）
  - 高峰日期分析
- 卡片指标：
  - 总进场人次
  - 今日进场人次
  - 本月日均进场
  - 单日最高进场
- 详细表格：进场编号、会员姓名、进场方式、关联卡号、进场时间、日期
- 时间筛选

#### ⑥ 员工佣金（保留原功能）
- 原 `CommissionFrame` 保留不动
- 如有需要可微调

#### ⑦ 课程安排（保留原功能）

## 五、核心统计数据逻辑

### 5.1 售课到期时间展示
```python
# 从售课记录中获取字段（已有字段，只需展示）：
#   "有效期截止日"、"课时有效期(天)"
# 计算剩余天数：
remaining_days = (expiry_date - today).days
# 状态标记：
if remaining_days < 0:
    status = "已过期"
elif remaining_days <= 7:
    status = "即将到期"
else:
    status = "正常"
```

### 5.2 课程包业绩统计
```python
# 数据源：lesson_package sheet
all_packages = self.biz.get_all_packages()

# 统计：
total_packages = len(all_packages)          # 已售包数
total_lessons = sum(p.get("总课时", 0))      # 总课时数
total_consumed = sum(p.get("已消耗课时", 0))  # 已消耗课时
total_remaining = total_lessons - total_consumed  # 剩余课时
active_packages = [p for p in all_packages if p.get("状态") == "有效"]
```

### 5.3 会籍卡业绩统计
```python
# 数据源：membership sheet
all_cards = self.biz.get_all_memberships()

# 按卡类型分组
type_stats = {}
for c in all_cards:
    card_type = c.get("卡类型", "")
    if card_type not in type_stats:
        type_stats[card_type] = {"count": 0, "amount": 0}
    type_stats[card_type]["count"] += 1
    type_stats[card_type]["amount"] += float(c.get("实收金额", 0) or 0)

# 时间筛选：按"出售日期"字段
```

### 5.4 会员进场统计
```python
# 数据源：checkin sheet
all_checkins = self.biz.get_all_checkins()

# 统计：
total_checkins = len(all_checkins)
today_checkins = self.biz.get_today_checkin_count()

# 按进场方式分组
category_stats = {}
for c in all_checkins:
    cat = c.get("进场方式", "")
    category_stats[cat] = category_stats.get(cat, 0) + 1

# 按小时统计时段分布
hour_distribution = [0] * 24
for c in all_checkins:
    time_str = c.get("进场时间", "")
    if ":" in str(time_str):
        hour = int(str(time_str).split(":")[0])
        if 0 <= hour <= 23:
            hour_distribution[hour] += 1
```

## 七、改造风险和兼容性

### 兼容性
- ✅ 不改动任何现有 Sheet 结构
- ✅ 不改动任何现有数据字段
- ✅ 不删除原有 stats_frame.py（可保留为历史，或拆除后删除）

### 风险点
1. **导航区长度增加**：导航栏需要滚动查看，建议把业绩总览设为默认打开页
2. **性能**：4个新页面同时加载大量数据，建议只在切换到该页面时计算
3. **原有 SaleStatsFrame 的增强**：需要同时保留期筛选和时间统计逻辑，只增加到期展示
