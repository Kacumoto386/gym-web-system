# 开发日志 V3.2.3
## 进场核销 新增进场按钮修复

**日期**: 2026-05-12 16:00
**版本**: v3.0.1 → v3.2.3
**仓库**: projects/gym-web-system

### 问题
进场核销页面中，"手动搜索"和"扫码/刷卡"两个按钮点了没有效果。原因是这两个按钮只是弹窗**内部**的模式切换（`switchCheckinMode`），但弹窗本身是 `hidden` 状态，没有任何按钮调用 `openCheckinModal()` 打开弹窗，导致点击后无视觉反馈。

### 修复
| 按钮 | 旧行为 | 新行为 |
|------|--------|--------|
| **➕ 新增进场**（新增） | — | 调用 `openCheckinModal()` 打开弹窗 |
| **🔍 手动搜索** | 仅切换弹窗内模式 | `openCheckinModal()` + 切换到手动模式 |
| **📱 扫码/刷卡** | 仅切换弹窗内模式 | `openScanCheckinModal()` 直接打开扫码模式 |

### 文件变更
| 文件 | 操作 | 说明 |
|------|:----:|------|
| `frontend/templates/checkin.html` | 修改 | 页面顶部新增"新增进场"按钮；修复模式按钮行为 |
| `backend/app.py` | 修改 | 版本号 3.2.2 → 3.2.3 |

### 验证

- ✅ 手动搜索模式弹窗默认激活
- ✅ 扫码/刷卡模式弹窗直接切换到扫码界面
- ✅ API 会员搜索正常
- ✅ 进场记录表格正常加载
