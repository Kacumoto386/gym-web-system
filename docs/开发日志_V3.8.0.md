# 开发日志 V3.8.0 — AI 助手循环死锁修复 + 批量余额查询

**日期**: 2026-05-23  
**类型**: fix + feat

---

## 变更内容

### 1. 修复 AI 助手逐个查询会员余额的无限循环死锁

**根因**：AI 助手查询会员数据时，对 1732 名会员逐个调用 `get_member_balance()`，但系统最多允许 10 轮工具调用，导致始终超时并反复重试，形成死循环。

**修复**：
- 删除单查工具 `get_member_balance`，新增统一工具 `get_member_balances`，不传参数时一次返回全部会员余额
- 改用 LEFT JOIN + GROUP BY 单条 SQL 替代 N+1 查询，性能提升明显
- 新增循环保护机制：同一工具调用 ≥3 次注入系统提示，≥5 次强制终止
- 系统提示规则第 6 条明确告知 `get_member_balances()` 不传参即可获取全部数据

### 2. 修复 _restart.py 在 MSYS2 环境下杀进程失败

**根因**：`taskkill /f /im uvicorn.exe` 在 MSYS2 bash 中 `/f` 被转换为 Windows 路径（F:/），导致杀进程静默失败，旧代码始终不更新。

**修复**：改用 `cmd.exe /c taskkill /f /im uvicorn.exe` 绕过 MSYS2 路径转义。
