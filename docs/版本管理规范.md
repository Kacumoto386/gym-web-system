# 鼠小弟健身管理系统 版本管理规范

## 版本号命名规则

采用 **主版本.次版本.修订号** 三段式：

```
V{MAJOR}.{MINOR}.{PATCH}
```

| 段位 | 含义 | 示例 |
|------|------|------|
| MAJOR | 重大架构变更、不兼容的 API 改动 | V3.0.0（桌面→网页） |
| MINOR | 新增功能、模块、页面，向后兼容 | V3.1.0（新增功能） |
| PATCH | Bug 修复、小优化、文档更新 | V3.0.1（修复登录循环） |

## 版本记录格式

每次版本变更需记录以下信息：

```
### YYYY-MM-DD HH:MM - V{version} {版本标题}

**类型**: {feature|fix|refactor|docs|chore}

**变更内容**:
{简要描述本次变更}

**文件变更清单**:
| 文件 | 操作 | 说明 |
|------|:----:|------|
| {文件路径} | {新增/修改/删除} | {变更说明} |

**Bug 修复清单**:
1. {Bug 描述}
2. {Bug 描述}

**数据迁移**: {是否需要迁移数据}
**回滚方案**: {回滚步骤}
```

## 分支策略

```
main            ← 稳定发布版
  ├── dev       ← 主开发分支
  ├── feat/*    ← 功能分支
  ├── fix/*     ← 修复分支
  └── release/* ← 发布准备分支
```

- **main**: 只接受 PR，不直接提交
- **dev**: 日常开发合并目标
- **feat/xxx**: 新功能，完成后 PR → dev
- **fix/xxx**: Bug 修复，完成后 PR → dev → main

## 提交信息规范

```
<类型>(<范围>): <简短描述>

<body>
```

| 类型 | 使用场景 |
|------|---------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档更新 |
| refactor | 重构 |
| chore | 杂项（构建、配置等） |
| style | 格式调整 |
| perf | 性能优化 |

示例：
```
feat(auth): 登录接口写入 cookie，修复循环重定向 fix(login): 修复管理员初始化失败。
```

### V3.2.1 — 2026-05-09 20:12
编辑会员保存修复 + 出生日期修复 + 年龄自动计算

**类型**: fix

**变更内容**:
修复编辑会员无法保存的问题（含/不含出生日期），增加年龄自动计算

**Bug 修复清单**:
1. 编辑表单无法保存（不填生日时）→ fetch 不带 cookie 导致 JWT 中间件拦截，加 `credentials: 'include'`
2. 编辑表单填写生日后 500 错误 → `MemberOut.birth_date` 标注为 `Optional[str]` 但数据库返回 `date` 对象，Pydantic v2 序列化校验失败；改为 `Optional[date]`
3. `update_member()` 中 `birth_date` 直接塞 `update_data` 字典 → SQLAlchemy 接收 `str` 写 `Date` 字段报类型错误；改为先 `date.fromisoformat()` 转换再单独赋值
4. 另 3 个 Pydantic schema 的 `birth_date` 也一并从 `Optional[str]` 修复为 `Optional[date]`

**优化**:
- 编辑弹窗出生日期旁增加**年龄自动计算**（填日期自动显示几岁）
- 新增弹窗同样加年龄显示
- `saveEditMember` 改用 `fetch` + `credentials: 'include'` 确保认证通过
- 优先用 `currentEditMemberId` 变量获取会员 ID，防止 DOM 读取异常

**文件变更清单**:
| 文件 | 操作 | 说明 |
|------|:----:|------|
| `backend/app.py` | 修改 | 版本号 3.2.0→3.2.1 |
| `backend/routers/member.py` | 修改 | `MemberOut` 及 3 个 schema 的 `birth_date` 类型修复；`update_member` 拆出生日处理；更新版本号 |
| `frontend/templates/members.html` | 修改 | 编辑表单改为 `fetch`+`credentials` 提交；新增 `saveEditMember`/`calcAge`/`updateEditAge`/`updateAddAge` 函数；新增年龄输入框 |

**数据迁移**: 无需
**回滚方案**: 回滚 member.py 和 members.html 到 V3.2.0 版本

## 发布流程

1. 从 dev 创建 release/v{version} 分支
2. 代码审查 + 测试
3. 更新版本号（app.py + 数据库）
4. 创建备份
5. 合并到 main
6. 打 tag: v{version}
7. 更新方案文档 + MEMORY.md
