# V2.15.7 进场核销UI优化方案

## 版本信息
- **版本**: v2.15.7
- **日期**: 2026-05-08
- **类型**: UI 优化 + Bug修复
- **前序版本**: v2.15.6 (自定义系统名称)

## 改动背景
首页看板「会员进场」面板在实际使用中存在两个问题：
1. 选择次卡/期限卡时，没有直观显示扣除金额或扣除次数
2. 上课教练下拉仅筛选了"教练"职务，无法选择门店管理员工

## 改动清单

| 文件 | 操作 | 说明 |
|------|:----:|------|
| `config.py` | 修改 | 版本号 2.15.6 → 2.15.7 |
| `gui/checkin_frame.py` | 修改 | 进场面板 UI 优化（见下方详细改动） |

### checkin_frame.py 详细改动

1. **跟进员工（原上课教练）**
   - `_load_coaches()` → `_load_all_staff()`
   - 加载所有在职员工（不限职务），含店长、顾问等
   - UI 标签改为「跟进员工」
   - 过滤条件：`员工状态` 为 "在职" / 空 / None

2. **卡片详情提示（新增区域）**
   - `card_detail_var` + 橙色标签
   - 次卡：显示剩余次数 + 有效期
   - 现金卡：显示余额
   - 期限卡：显示有效期区间
   - 触发：核销方式切换、卡片选择变更均刷新

3. **扣减预览（新增区域）**
   - `deduct_preview_var` + 红色标签
   - 次卡：实时显示扣除后剩余次数（不足时警告）
   - 现金卡：实时显示扣除后余额（不足时警告）
   - 期限卡/无卡进场：显示"不扣费"
   - `qty_var` 绑定 `trace_add("write")` 变化时实时刷新

4. **核销方式与会籍卡双向同步**
   - 选择会籍卡 → 自动推断卡类型 → 同步切换核销方式
   - 切换核销方式 → 自动选中同类型会籍卡
   - 消除核销方式与卡类型不匹配的问题
