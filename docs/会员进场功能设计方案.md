# 会员进场（签到核销）功能设计方案

## 1. 功能概述

在首页看板顶部增加一个**「会员进场」大按钮**，门店前台可通过该功能：
1. 输入/扫描会员编号快速定位会员
2. 查看会员名下的 **有效会籍卡**（次卡余量 / 现金卡余额 / 期限卡状态）
3. 选择消费方式：**扣次卡次数** 或 **扣现金卡余额**
4. 完成核销后自动记录为一笔「进场记录」+ 联动系统各项数据

### 核心价值
- 门店日常最频繁操作，需要极致快捷
- 不需要经过"会员管理→售课→上课"的复杂流程
- 选择会籍卡类型后 **一键核销**，半分钟内完成

## 2. 数据流设计

```
会员进场
  │
  ├─ 输入会员编号（扫码枪/手动输入）
  │
  ├─ 查询有效会籍卡
  │    ├─ 次卡：显示剩余次数
  │    ├─ 现金卡：显示余额
  │    └─ 期限卡：显示有效期（不限次则直接通过）
  │
  ├─ 选择核销方式
  │    ├─ 次卡 → 扣 1 次 → 自动创建上课记录（签到模式）
  │    ├─ 现金卡 → 扣 ¥X（可输入）→ 自动创建上课记录
  │    └─ 期限卡 → 仅签到，不扣减 → 自动创建上课记录
  │
  └─ 完成进场
       │
       ├─ 写入「进场记录」Sheet（新）
       │    ├─ 进场编号、会员信息、核销方式、扣减数量
       │    ├─ 员工/门店、进场时间
       │    └─ 操作员工
       │
       ├─ 自动创建上课记录（签到模式）
       │    ├─ 课程名称 = "进场签到"（或可输入）
       │    ├─ 课耗课时 = 1
       │    ├─ 上课教练 = 操作员工（可选）
       │    └─ 签到状态 = "已签到"
       │
       ├─ 更新会籍卡状态（次卡扣次数/现金卡扣余额）
       │
       └─ 更新首页看板指标（今日进场人数等）
```

## 7. 业务规则

### 7.1 核销逻辑

| 卡类型 | 进场规则 | 扣减影响 |
|--------|----------|----------|
| **次卡** | 每次进场扣 1 次（可配置） | 会籍卡.剩余次数-1；会员.剩余课时-1 |
| **现金卡** | 每次进场扣指定金额（前台输入） | 会籍卡.余额-¥X；已消费金额+¥X |
| **期限卡** | 不限次签到，仅记录进场 | 不扣减任何次数/金额 |

### 7.2 会籍卡优先级选择
- 会员持有 **多张同类会籍卡**（如 2 张次卡）时：
  - 优先消耗 **最早开卡** 的 → 到期时间最近的
  - 前台也可手动选择使用哪张卡
- 会员同时持有 **异类会籍卡**（次卡+现金卡）时：
  - 前台在下拉框中选择核销方式

### 7.3 自动创建上课记录
核销成功后自动创建上课记录（class_record），字段映射：

| 上课记录字段 | 值 |
|-------------|-----|
| 上课编号 | 自动生成（CR+日期+序号） |
| 会员编号 | 会员编号 |
| 会员姓名 | 会员姓名 |
| 课程名称 | "进场签到"（前台可修改） |
| 上课日期 | 当天日期 |
| 上课时段 | 当前时间 |
| 课耗课时 | 1（次卡）/ 0（期限卡）/ 0（现金卡） |
| 签到状态 | "已签到" |

### 7.4 前台界面规则
1. **扫码枪支持** — 输入框自动获取焦点，回车即触发查询
2. **成员卡片展开** — 查到一个会员后，展开显示其所有有效会籍卡
3. **核销确认** — 确认时弹 brief 确认框，减少误操作
4. **清空/重置** — 核销成功后自动清空，准备下一次进场

## 9. 关键交互细节

### 扫码枪输入流
```
光标在输入框（AutoFocus）
  │
  ├─ 手动输入编号 + Enter
  │    └─ 立即查询会员，展开卡片
  │
  ├─ 扫码枪扫描  →  自动填充 + 触发查询
  │    └─ 类似 Enter 键行为
  │
  └─ 输入不存在的会员
       └─ 提示"未找到会员"，可选择新增会员或重试
```

### 核销确认弹窗
```
┌───────────────────────┐
│ ✅ 确认进场            │
│ ─────────────────────  │
│ 会员：张三              │
│ 核销：30次卡 × 1       │
│ ├ 剩余 28次 → 27次     │
│ ─────────────────────  │
│  [✅ 确认] [❌ 取消]   │
└───────────────────────┘
```
