# 开发日志

## 2026-05-09 12:47 - V3.1.7 业绩数据展示修复

### 问题
业绩统计页面显示原始 JSON 数据而非渲染好的 HTML 卡片/表格。

### 根因
业绩统计模块的 API 端点全部返回 JSON 格式（`{"cards": [...], "html": "<div>..."}`），但 HTMX 需要直接接收 HTML 片段。FastAPI 自动将纯字符串返回值序列化为 JSON 字符串（外层包双引号），导致页面显示原始数据。

### 修复
在 `backend/routers/performance.py` 中，将所有被 HTMX 加载的端点改为 `return HTMLResponse(content=...)`：

**8 处修复覆盖 12 个端点：**
- 统计卡片端点 (6个): `overview/cards`, `sales/stats`, `packages/stats`, `cards/stats`, `checkins/stats`, `checkins/by-type`
- 时段分布端点 (1个): `checkins/by-hour`
- 表格端点 (4个): `sales/table`, `packages/table`, `cards/table`, `checkins/table`
- 类型分布端点 (1个): `cards/by-type`

### 受影响文件
- `backend/routers/performance.py` — 8 处 return 语句改为 `HTMLResponse()`
- `backend/app.py` — 健康检查版本号同步为 3.1.7
