# 开发日志 V3.9.0 — 会员密码+验证码登录 + 小程序版号 M1.0.1

**日期**: 2026-05-27  
**类型**: feat

---

## 变更内容

### 1. 会员密码 + 验证码登录系统

**背景**: 会员小程序仅支持微信 code 登录（mock 阶段直接输手机号），没有密码验证，不安全且不符合实际需求。

**实现**:

#### 数据库
- `Member` 模型新增 `password_hash` 字段（VARCHAR(200)，bcrypt 哈希）
- `init_db()` 添加 ALTER TABLE 迁移（SQLAlchemy 2.0 `with engine.connect()` 模式）

#### 后端 API（4 个新端点）

| 端点 | 功能 |
|------|------|
| `POST /auth/send-code` | 发送验证码（开发阶段直接返回 code） |
| `POST /auth/login-by-password` | 手机号 + bcrypt 密码 → JWT |
| `POST /auth/login-by-code` | 手机号 + 短信验证码 → JWT |
| `POST /auth/set-password` | 验证码验证身份后设置/重置密码 |

- 验证码存储于内存 dict（开发阶段用，生产环境需替换为 Redis）
- 复用 `backend/routers.auth.hash_password()` / `verify_password()` bcrypt 函数
- 保留原有微信 code 登录端点，向后兼容

#### 前端（微信小程序）
- 登录页重构为双 Tab：密码登录 / 验证码登录
- Tab 切换栏，底部高亮指示
- 密码登录：手机号 + 密码输入 + 校验
- 验证码登录：手机号 + 验证码输入 + 60s 倒计时按钮
- 获取验证码前校验手机号是否已注册
- 保留微信手机号一键登录按钮

### 2. 会员小程序版号 M1.0.1

- `app.js` globalData 新增 `version: 'M1.0.1'`
- 首页底部显示版本号 `会员小程序 vM1.0.1`

```
POST /auth/send-code             → {"code":0, "data":{"code":"271215"}}       ✅
POST /auth/login-by-code         → JWT + 会员信息                            ✅
POST /auth/set-password          → "密码设置成功"                            ✅
POST /auth/login-by-password     → JWT + 会员信息                            ✅
POST /auth/login-by-password(错) → "手机号或密码错误"                        ✅
POST /auth/send-code(未注册手机) → "该手机号未注册"                          ✅
POST /auth/login-by-code(错码)   → "验证码错误"                              ✅
POST /auth/login(原微信登录)     → 仍正常工作                                ✅
```
