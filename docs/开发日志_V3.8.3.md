# 开发日志 V3.8.3 — 数据导入增强（会员去重 / 会籍卡 / 私教课导入）

**日期**: 2026-05-25  
**类型**: feat

---

## 变更内容

### 1. 会员导入模板优化

**改动**：
- 删除「开卡日期」「到期日期」两个字段（导入意义不明确）
- 调整模板字段顺序使其更简洁

### 2. 删除课程导入模块

**改动**：
- 移除 `IMPORT_FIELDS` 中的 `course` 类型配置
- 前端删除「课程导入」Tab
- 课程数据由系统管理，无需批量导入

### 3. 新增「已售会籍卡」导入

**改动**：
- 新增 `membership_card` 类型配置（`IMPORT_FIELDS` + `TYPE_CONFIG` + `DATE_FIELDS` + `DECIMAL_FIELDS` + `PROTECTED_FIELDS`）
- 模板字段：会员手机号*、会员姓名*、会籍类型、卡名称、有效期(天)、售价、有效期起、有效期止、状态、备注
- 导入时自动生成 `card_id`（前缀 `MC`），设置 `is_product=0`
- 自动跳过 `member_phone` 字段（MembershipCard 表无此列）

### 4. 新增「已售私教课」导入

**改动**：
- 新增 `sold_lesson` 类型配置
- 模板字段：会员手机号*、会员姓名*、课程名称、售课日期、购买课时、赠送课时、总课时、单价、总价、实收金额、付款方式、销售员工编号、备注
- 导入时自动生成 `sale_id`（前缀 `SL`）
- 保留 `member_phone` 字段（Sale 表有此列）

### 5. 会员导入：严格防重复机制

**改动**：
- 后台执行 `_run_import` 中，`import_type == "member"` 时整批预检
- 手机号/姓名任一重复 → 整批拒绝，状态置为 `failed`
- 前端显示批处理错误信息（row=1 的 batch 级别错误）
- 仅针对会员导入模块，不影响其他导入类型

### 6. 会籍卡/私教课导入：会员有效性校验

**改动**：
- 上传解析时，根据 `member_phone` 查询 `Member` 表
- 手机号对应的会员不存在 → 该行报错跳过
- 会员姓名不匹配系统记录 → 报错跳过
- 校验通过后注入 `member_id` 供后续 INSERT 使用

### 7. 前端 Tab 更新

**改动**：
- `import.html` 移除「课程导入」Tab
- 新增「会籍卡导入」「私教课导入」Tab
- Tab 切换时自动更新下载链接和隐藏类型输入
