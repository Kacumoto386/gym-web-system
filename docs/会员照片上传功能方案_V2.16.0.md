# V2.16.0 会员照片上传功能方案

## 版本信息
- **版本**: v2.16.0
- **类型**: 新功能
- **前序版本**: v2.15.8 (教练排班日历化改造)

## 需求
在会员信息管理（新增/编辑对话框）中增加会员照片功能：
1. **拍照** — 调用电脑摄像头拍照保存（tkinter 内嵌实时预览窗口）
2. **本地上传** — 从本地选择图片文件
3. **删除** — 移除照片文件并清空 Excel 路径

## 存储方案

### 文件存储（非Excel）
Excel 不适合存二进制图片。采用**文件系统存储 + Excel 存路径**的方案：

```
data/member_photos/
  └── M20260508001.jpg   ← 会员编号.jpg（统一为 JPG 格式）
```

会员信息表新增列：`照片路径`（存相对路径如 `member_photos/M20260508001.jpg`）

### 优势
- Excel 文件大小不受影响
- 图片集中管理，方便备份/迁移
- 读取效率高（只读路径，不读二进制）

## 拍照窗口设计

改用纯 tkinter 实现（避免 OpenCV 原生窗口阻塞问题）：

```
┌─────────────────────────────┐
│  拍照 - 确认拍照 / 取消      │
├─────────────────────────────┤
│                             │
│   ┌─────────────────────┐   │
│   │    实时摄像头预览     │   │
│   │    (30ms 刷新)       │   │
│   └─────────────────────┘   │
│                             │
│     [📷 拍照]    [❌ 取消]   │
└─────────────────────────────┘
```

- 实时预览使用 `win.after(30, update_frame)` 循环，不阻塞 tkinter 主循环
- 点击拍照捕获最新帧，关闭窗口返回 PIL Image
- 取消或关闭窗口直接退出，返回 None
- 窗口使用 `grab_set()` 模态，`wait_window()` 阻塞等待

### 依赖策略
- **Pillow**：必需。用于加载、缩放、圆形裁剪、保存照片

- 用户运行前需 `pip install Pillow`，拍照需额外 `pip install opencv-python`

## 数据流

```
用户点击"拍照"按钮
  → _on_capture()
    → capture_from_camera()
      → 弹出 tkinter Toplevel 实时预览
      → 用户点击"拍照" → 捕获帧 → 关闭窗口
      → 返回 PIL Image
    → _display_photo(img) → 圆形裁剪 → Canvas 显示

用户点击"保存"按钮
  → on_save()
    → 保存表单数据到 Excel
    → 如果有照片 → save_photo(member_id, current_photo_pil)
      → 缩放裁剪为 200×200 正方形
      → 写入 data/member_photos/{member_id}.jpg
    → update_member_photo_path(row_num, rel_path)
      → 写入 Excel「照片路径」列
    → 关闭对话框
```

## 关键设计决策

### 1. 圆形头像实现
使用 PIL 的 RGBA 模式 + 圆形蒙版（ImageDraw.ellipse 绘白色椭圆）+ paste with mask → PhotoImage

### 2. 照片文件名统一为 jpg
无论上传的图片是 png/gif/bmp，统一保存为 JPEG 格式，减少格式兼容问题

### 3. 保存时裁剪正方形
对上传/拍摄的图片取短边中心裁剪为正方形，再缩放到 200×200，确保头像显示一致

### 4. 兼容旧数据
已有会员照片路径为空 → 显示默认 👤 占位
Excel 未初始化「照片路径」列 → 自动在首次访问时创建

## Bug 修复记录

### Bug 1: 拍照窗口无法关闭/保存
**问题**：V1 实现使用 OpenCV 原生窗口 (cv2.imshow + cv2.waitKey)，在 tkinter 中阻塞主循环导致窗口无法操作
**修复**：V2 改用纯 tkinter Toplevel + after 循环刷新帧实现拍照预览

### Bug 2: 会员删除报错
**问题**：`on_delete` 传了 2 个参数 (row_num, member_id)，但后端 `delete_member` 只接受 1 个
**修复**：后端增加可选 `member_id` 参数，同时自动清理对应照片文件

## 兼容性
- 已有会员照片路径为空 → 显示默认 👤 占位
- 无 opencv → `capture_from_camera()` 弹出提示"摄像头模块未安装"
- Pillow 必须安装，否则整个功能不可用

## 注意事项
1. 如果用户没有摄像头或 opencv，拍照按钮会弹出提示而不会崩溃
2. 删除会员会同时清理照片文件（如果传入了 member_id）
3. 照片路径存为相对路径，确保项目目录移动后仍可找到
