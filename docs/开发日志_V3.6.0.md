# 开发日志 V3.6.0 — 售课到期日期显示修复

**日期**: 2026-05-22  
**类型**: bugfix + enhancement  
**触发**: 用户反馈录入购课数据后"到期"列不显示日期

---

## 背景

售课记录列表中，"到期"列始终显示 `-`。`_build_table()` 调用 `_get_lesson_info()` 仅从 `LessonPackage` 表查询 `valid_until`，但创建售课记录时不会自动创建 `LessonPackage` 记录，导致查询结果为空。

同时，售课录入表单在选择课程后没有自动填入到期日期，需要用户手动计算并填写。

---

## 修复详情

### 1. `_get_lesson_info()` fallback 到 sale.end_date

**文件**: `backend/routers/sale.py:63-68`

```python
if remaining is None:
    # 无对应 LessonPackage 时回退到 Sale 自身的到期时间
    return sale.total_hours or 0, sale.total_hours or 0, sale.end_date
# 即使有包但无到期时间，也回退到 Sale 自身的到期时间
if valid_until is None and sale.end_date:
    valid_until = sale.end_date
```

无 `LessonPackage` 数据时回退到 `sale.end_date`，避免显示 `-`。

### 2. 课程 search-json 增加 valid_days

**文件**: `backend/routers/course.py:276-278`

`/api/courses/search-json` 端点返回课程信息时增加 `valid_days` 字段，供前端计算到期日期使用。

### 3. 售课表单自动计算到期日期

**文件**: `frontend/templates/sales.html`

- `selectCourse()`: 选择课程时根据 `start_date + validDays` 自动填入到期日期
- `autoCalcEndDate()`: 修改开卡日期时自动重算到期日期（`onchange="autoCalcEndDate()"`）

### 4. 服务器缓存问题

排查过程中发现服务运行的是旧的字节码缓存（`__pycache__`），导致代码修改未生效。清除缓存并重启后修复。
