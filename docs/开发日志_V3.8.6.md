# 开发日志 V3.8.6 — 会籍卡实收金额 / 销售员手机号匹配

**日期**: 2026-05-25  
**类型**: feat

---

## 变更内容

### 1. 会籍卡导入增加实收金额（必填）

**背景**: 会籍卡导入缺少实收金额字段，系统无法区分标价和实际收款，财务报表统计不准确。

**改动**：
- MembershipCard 模型新增 `actual_amount` 列（DECIMAL(10,2)，实收金额）
- `database.py` 增加 ALTER TABLE 自动迁移
- IMPORT_FIELDS 中 `membership_card` 新增 `actual_amount*`（必填字段），位于 `price` 之后
- `DECIMAL_FIELDS` 新增 `actual_amount`
- 提示行示例值：`"3000"`（与 price 同值示例）

### 2. 会籍卡导入增加销售员手机号

**背景**: 导入会籍卡时无法关联销售员，需手动补填。

**改动**：
- MembershipCard 模型新增 `staff_id`（销售员工编号）、`staff_name`（销售员姓名）列
- `database.py` 增加 2 条 ALTER TABLE 自动迁移
- IMPORT_FIELDS 中 `membership_card` 新增 `staff_phone`（销售员手机号）字段
- 上传时根据手机号查询 Staff 表，自动注入 `staff_id` + `staff_name`，员工不存在时报错
- `_create_record` 的 skip_fields 增加 `staff_phone`（不直接存储，仅用于查找）

### 3. 私教课导入字段替换

**背景**: 用户更习惯用手机号而非员工编号来匹配销售员。

**改动**：
- IMPORT_FIELDS 中 `sold_lesson` 的 `staff_id`（销售员工编号）替换为 `staff_phone`（销售员手机号）
- 上传时复用会籍卡相同的 Staff 手机号查找逻辑
- `_create_record` 的 skip_fields 增加 `staff_phone`
- 提示行示例值：`"13800138001"`（手机号格式）
