**日期**: 2026-05-22  
**类型**: feature

---

## 变更内容

### 1. 预约详情查看

预约管理页面所有状态行（已预约/已签到/已完成/已取消）的操作列新增"详情"按钮，点击后弹出模态框，展示预约完整信息：

- 预约编号 · 创建时间
- 预约日期 · 时间范围
- 会员姓名 · 会员电话
- 课程名称 · 课程编号
- 教练姓名 · 上课地点
- 预约状态 · 签到人数
- 手环编号 · 门店编号

**涉及文件**:
- `backend/routers/booking.py` — 操作列增加详情按钮；`get_booking()` 端点补充 `sign_in_count`、`store_id`、`wristband_id`、`created_at` 字段
- `frontend/templates/booking.html` — 新增详情模态框 + `viewBooking()` / `closeViewBooking()` 函数

### 2. 结束时间校验

新增预约时校验结束时间不能早于开始时间，前后端双重验证：

- **前端** (`submitBooking()`): 表单提交前比较 `startTime` 和 `endTime`，不合法时提示错误并阻止提交
- **后端** (`create_booking()`): API 层同样校验，防止绕过前端直接调用

## 验证方法

1. 预约管理 → 各状态行均显示"详情"按钮
2. 点击"详情" → 弹出模态框，显示完整预约信息
3. 新增预约 → 结束时间填早于开始时间 → 提示"结束时间必须晚于开始时间"
4. 新增预约 → 结束时间晚于开始时间 → 正常提交
5. 通过 API 直接提交不合法的结束时间 → 返回 400
