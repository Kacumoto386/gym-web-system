## 2026-05-09 修复记录

### V3.1.8-hotfix1: 期限卡签到 + 现金卡自定义扣费

**问题 1：期限卡签到选项不显示**

**现象**：选择期限卡（月卡/季卡/年卡/时卡）后，核销方式中不显示"期限卡签到"选项，只显示"储值扣款"和"无卡体验"。

**根因**：
- 数据库 `membership_card` 表的 `card_type` 字段存的是"期限卡"（大类名）
- 前端 JS 代码用 `['月卡','季卡','年卡','时卡'].includes(ctype)` 来判断是否属于期限卡
- `ctype` 实际值为"期限卡"，与上述列表中任何值都不匹配 → 判断结果为 false → 选项未被生成

**修复（3 处）**：

| 文件 | 行号 | 修改内容 |
|------|:----:|----------|
| `frontend/templates/checkin.html` | ~476 | `onCardSelect()` 中期限卡判断增加 `ctype === '期限卡'` |
| `frontend/templates/index.html` | ~184 | `has期限卡` 变量定义增加 `c.card_type === '期限卡'` |
| `frontend/templates/index.html` | ~291 | `showQuickOptionsFallback()` 中期限卡判断增加 `ctype === '期限卡'` |

**问题 2：现金卡扣费默认 ¥30**

**现象**：现金卡扣费预设 `value="30"`，用户无法自定义金额。

**修复文件与逻辑**：

| 文件 | 修改内容 |
|------|----------|
| `frontend/templates/checkin.html` | `onCardSelect()` 中现金卡 option 去掉 `value="30"`，改为 `placeholder="输入金额"` |
| `frontend/templates/checkin.html` | `onConsumeOptionChange()` 中现金卡扣费也显示金额输入框 |
| `frontend/templates/checkin.html` | `updateDeductPreview()` 改用 `cardSelect.value` 获取当前卡片，用输入框金额计算 |
| `frontend/templates/checkin.html` | `submitCheckin()` 从 `cardSelect` 获取 `card_id` 而非隐藏输入 |
| `frontend/templates/index.html` | 快速签到（主路径 + 回退路径）去掉 `value="30"` 和所有硬编码扣费金额 |
| `backend/routers/checkin.py` | `create_checkin()` 现金卡分支：`consume_amount` 必传，`<= 0` 时报错，余额不足时报错 |

**问题 3：浏览器缓存导致服务端代码更新不生效**

**现象**：服务端已更新代码，但浏览器加载旧版 HTML/JS。

**修复**：
- `backend/app.py` 中 `/` 和 `/checkin` 页面路由添加 `Cache-Control: no-cache, no-store, must-revalidate`
- 响应头：`Pragma: no-cache`、`Expires: 0`

### 验证方法

1. 选一个有效的期限卡会员 → 核销方式应出现"期限卡签到"
2. 选一个有效的现金卡会员 → 核销方式应出现"现金卡扣费" + 空的金额输入框
3. 现金卡扣费提交空金额 → 后端返回错误提示
4. 现金卡扣费提交超出余额金额 → 后端返回错误提示
5. 期限卡签到 → 成功签到，不扣减任何资源
