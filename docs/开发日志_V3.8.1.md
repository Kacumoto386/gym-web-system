# 开发日志 V3.8.1 — 会员充值模块增强

**日期**: 2026-05-23  
**类型**: feat

---

## 变更内容

### 1. 新增充值弹窗重构 — 会员搜索选择器

**改动**：
- 原手动输入会员编号/姓名字段改为搜索选择器
- 支持按姓名、手机号、会员编号实时搜索（调用 `/api/members/search-json`，300ms 防抖）
- 选中会员后自动填充隐藏的 `member_id` 和 `member_name`
- 未选择会员时提交会提示"请先选择会员"
- 打开弹窗自动填入当天日期

### 2. 新增「到期日期」字段

**改动**：
- 数据库 `recharge` 表新增 `expiry_date` (Date, nullable) 列
- 自动迁移：`init_db()` 中执行 `ALTER TABLE`（兼容已有数据库）
- 充值表单新增到期日期输入框
- Pydantic schema `RechargeOut.expiry_date` 用于 REST API 响应

### 3. 修复 Decimal/float 类型不匹配

**根因**：Pydantic v2 将表单金额解析为 `float`，但 SQLAlchemy `DECIMAL` 列返回 `Decimal`，Python 不支持 `Decimal + float`。

**修复**：创建充值时用 `Decimal(str(amount))` 统一转换后再做算术运算。

### 4. 修复 HTMX 表单提交 — Pydantic BaseModel 与 form-encoded 不兼容

**根因**：FastAPI + Pydantic v2 的 `BaseModel` 参数只接受 JSON body，但 HTMX 默认以 `application/x-www-form-urlencoded` 提交表单，导致始终返回 422。

**修复**：`create_recharge()` 改用 `Form(...)` 参数声明，与其他 HTMX 端点（booking/checkin/member）保持一致。

### 5. 修复保存后不刷新表格

**根因**：移除了 `hx-target` 后，HTMX 将 JSON 响应直接替换到 form 元素本身，破坏后续刷新逻辑。

**修复**：添加 `hx-swap="none"` 禁止 HTMX 自动替换内容，成功后在 `hx-on::after-request` 中手动调用 `htmx.ajax('GET', '/api/recharges/table?_='+Date.now(), ...)` 刷新表格。

### 6. 经办员工改为下拉选择

**改动**：
- 原文本输入改为 `<select>` 下拉框
- 打开弹窗时通过 `/api/staff/search-json` 加载在职员工列表（前端过滤 `status='在职'`）
- `/api/staff/search-json` 新增返回 `status` 字段

### 7. 充值表格新增两列

- **到期时间** — 显示充值设置的到期日期
- **剩余余额(含赠金)** — 实时查询会员当前总余额（带 ¥ 前缀），通过预加载 Member 避免 N+1
