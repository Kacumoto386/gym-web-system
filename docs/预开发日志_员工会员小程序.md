# 预开发日志 — 员工小程序 + 会员小程序 接口设计

**日期**: 2026-05-27  
**类型**: design, pre-dev  
**状态**: 已设计完成，待开工落实

---

## 概述

为系统扩展两套微信小程序端：

1. **员工小程序** — 前台/教练的移动销售工具，核心场景：会员录入 → 购卡 → 购课 → 预约 → 核销
2. **会员小程序** — 会员自助服务端，核心场景：会籍查询 → 预约课程 → 签到记录 → 体测查看

两套小程序共享现有 FastAPI 后端，新增 `backend/miniapp/` 模块，通过 `/api/miniapp/staff/*` 和 `/api/miniapp/member/*` 路径前缀与现有后台 API 隔离。

```
│   │   ├── class_record.py     # 上课记录
│   │   └── performance.py      # 业绩提成查询
│   └── member/

│       ├── balance.py          # 储值余额/充值记录
│       ├── checkin.py          # 我的签到记录
│       ├── booking.py          # 预约/取消预约

│       ├── body.py             # 体测记录
│       └── alert.py            # 到期提醒
```

### 2. 认证体系

| 身份 | 方式 | JWT Payload |
|------|------|-------------|
| Admin（现有） | Cookie + Username/Password | `sub: "admin"` |
| 员工小程序 | Bearer Token + Username/Password | `sub: staff_id, type: "staff"` |
| 会员小程序 | Bearer Token + 微信 code/手机号 | `sub: member_id, type: "member"` |

### 3. 数据库新增

- `staff_wechat_accounts` — 员工微信账号映射表
- `member_wechat_accounts` — 会员微信账号映射表
- `system_config`（可选）— 微信小程序配置存储

### 4. 响应格式统一

小程序接口统一返回 `{"code": 0, "data": ..., "message": "ok"}`，区别于后台多种响应风格。

错误码：
| 范围 | 含义 |
|------|------|
| 0 | 成功 |
| 1001-1002 | 认证异常（令牌无效/过期） |
| 2001-2002 | 参数异常（参数错误/资源不存在） |
| 3001 | 业务逻辑错误（余额不足等） |
| 5000 | 服务端内部错误 |

## 支付流程设计

```
选择商品 → 创建订单(pending) → 发起支付 → 用户确认 → 支付回调 → 激活权益
                                                           ↓
                                                 创建会籍卡/售课记录
```

### 订单状态

| 状态 | 说明 |
|------|------|
| pending | 待支付 — 订单已创建，锁住商品信息 |
| paid | 已支付 — 等待异步激活（预开发直接进入 completed） |
| completed | 已完成 — 权益已激活（会籍卡已开卡 / 课程包已生效） |
| cancelled | 已取消 |

### 预开发阶段 vs 正式上线

| 环节 | 预开发阶段 | 正式上线 |
|------|-----------|---------|
| 商品展示 | 从 Product/Package 表读取 | 不变 |
| 创建订单 | 订单存入内存 dict | 迁移到 purchase_orders 表 |
| 发起支付 | 返回模拟 prepay_id | 调微信支付 JSAPI 统一下单 |
| 确认支付 | 直接调 `/purchase/pay` 模拟成功 | 微信异步回调 `/purchase/callback` |
| 激活权益 | 同步执行（_activate_order） | 回调中执行 |

### 预需数据库新增

```sql
-- 订单表（正式版迁移）
CREATE TABLE purchase_orders (
    order_id VARCHAR(30) PRIMARY KEY,
    member_id VARCHAR(20) NOT NULL REFERENCES members(member_id),
    product_type VARCHAR(10) NOT NULL,        -- card / package
    product_id VARCHAR(30) NOT NULL,
    product_name VARCHAR(200) NOT NULL,
    quantity INTEGER DEFAULT 1,
    amount DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',     -- pending / paid / completed / cancelled
    product_meta TEXT,                         -- JSON: 激活权益所需额外信息
    remark TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    paid_at TIMESTAMP,
    wx_transaction_id VARCHAR(100)             -- 微信支付交易号
);
```

## 技术决策

1. **双通道 JWT** — 与后台共用签名密钥，Payload 增加 `type` 字段区分 admin/staff/member
2. **业务逻辑复用** — 第一阶段在 miniapp 路由中直接查询 DB（与现有 router 风格一致），后续抽取 service 层可选
3. **零改动现有代码** — 不修改 `routers/` 下现有路由函数，通过 `features.yaml` 开关控制小程序模块注册
4. **统一错误处理** — `MiniAppException` 异常类 + FastAPI `exception_handler` 全局捕获

## 实施路径

```
第一优先级 → 认证体系 + 员工(录入/购卡/核销/查询) + 会员(会籍/签到/预约)
第二优先级 → 员工(购课/预约/上课/提成) + 会员(体测/上课/储值/到期)
第三优先级 → 消息通知 + 数据看板 + 签到统计 + 性能优化
```
