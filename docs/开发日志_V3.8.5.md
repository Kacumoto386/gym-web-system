# 开发日志 V3.8.5 — 会籍卡导入增加卡种字段 + 私教课课程种类

**日期**: 2026-05-25  
**类型**: feat

---

## 变更内容

### 1. 会籍卡导入增加卡种字段

**背景**: 原会籍卡导入仅支持基本字段（售价、有效期、状态等），次卡/现金卡等特殊卡种的关键字段缺失，导入后需手动补充。

**改动**：
- IMPORT_FIELDS 中 `membership_card` 新增 4 个字段：
  - `total_classes`（购买次数）— 次卡专用，记录购买的课时次数
  - `bonus_classes`（赠送次数）— 次卡专用，记录赠送的课时次数
  - `face_value`（面值/余额总额）— 现金卡专用，记录卡面值
  - `consumed_amount`（已消费金额）— 现金卡专用，记录已扣减金额
- `consumed_amount` 从 PROTECTED_FIELDS 中移除，允许导入时写入
- `DECIMAL_FIELDS` 中 `membership_card` 新增 `face_value`、`consumed_amount`

### 2. 会籍卡导入卡种联动 — 会员资产更新

**改动**：
- 导入会籍卡时，根据 `card_type` 自动更新会员资产：
  - **次卡**: `total_classes + bonus_classes` 累加到 `Member.remaining_lessons`
  - **现金卡/现金**: `price` 累加到 `Member.balance`

### 3. 私教课导入增加课程种类

**背景**: 私教课导入缺少课程种类字段，无法区分私教课/体验课/小班课/团课。

**改动**：
- Sale 模型新增 `course_type` 列（VARCHAR(50)），可选值：私教课/体验课/小班课/团课
- `database.py` 增加 `ALTER TABLE sale ADD COLUMN course_type` 自动迁移
- IMPORT_FIELDS 中 `sold_lesson` 新增 `course_type`（课程种类）字段
- 提示行示例值：`"私教课"`

### 4. 导入 Tab 切换 Bug 修复

**改动**：
- `switchType()` 函数中移除 `tab.dataset.type = type;` 行 — 该行在 forEach 循环中将所有 tab 的 `data-type` 改为当前选中值，导致第二次点击时高亮失效
