## V3.0.0 数据迁移阶段

| 分类 | 详情 |
|------|------|
| **版本** | V3.0.0 |
| **日期** | 2026-05-08 18:16 |
| **涉及文件** | `backend/migrate_from_excel.py` |

### Bug 1: 空 ID 自动生成

**现象**: 迁移过程中部分记录 `_id` 字段为空，导致 SQL 唯一约束冲突。

**根因**: Excel 系统中部分记录的编号字段为空，SQLite 迁移时直接照搬。

**修复**: `migrate_from_excel.py` 增加自动检测模型 unique `_id` 字段的逻辑 — 若检测到空值则调用 `generate_id()` 自动填充。

### Bug 2: 字段名不匹配

**现象**: 收入总账表的日期字段名为"日期"而非预期的"收入日期"，支出表的日期字段名为"日期"而非"支出日期"。

**根因**: 原 Excel 表头命名与 ORM 模型字段不一致。

**修复**: 在迁移脚本中根据表名做特殊字段映射，读取"日期"列写入对应的 ORM 日期字段。

| 分类 | 详情 |

| **日期** | 2026-05-08 18:49 |
| **涉及文件** | `backend/routers/member.py`, `backend/routers/checkin.py` |

### Bug: TemplateResponse 参数顺序错误

**现象**: 访问会员管理、课程管理等页面时抛出 `jinja2.exceptions.TemplateNotFound`，但模板文件实际存在。

**根因**: 对 `TemplateResponse(request, name, context)` 的调用，前两参数应为 `(request, template_name)`，实际传成了 `(template_name, request)`。

**修复**: 修正所有 `TemplateResponse` 调用的参数顺序。

### Bug: create_checkin 使用 Query 参数

**现象**: HTMX 默认使用 `application/x-www-form-urlencoded` 提交表单，但后端 `create_checkin` 用 `Query` 而非 `Form` 接收参数。

**根因**: 参数声明方式与 HTMX 提交的数据编码格式不匹配。

**修复**: 将 `Query(...)` 改为 `Form(...)`。

## V3.0.1 登录系统

| 分类 | 详情 |
|------|------|
| **版本** | V3.0.1（热修复） |
| **日期** | 2026-05-08 20:26 |
| **涉及文件** | `backend/routers/auth.py` |
| **备份** | `projects/_backups/gym-web-system_v3.0.1_20260508_2032/` |

### Bug: 登录后无限循环重定向

**现象**: 登录成功后跳转首页 → 立刻又被重定向回登录页 → 死循环。浏览器无法访问任何页面。

**根因**: 
- `POST /auth/token` 只返回 `JSON` 格式的 token（`{"access_token": "xxx"}`）
- 前端 JS 收到 JSON 后 `window.location.href = '/'` 跳转首页
- 但认证中间件从 cookie 读取 token，而 cookie 中没有 token
- 中间件检测到无 token（"未登录"）→ 302 重定向到 `/auth/login`
- 循环重复

**修复**: `POST /auth/token` 在返回 JSON 的同时，通过 `Set-Cookie` 响应头将 JWT token 写入 `access_token` cookie（HttpOnly, 24h 有效期, path=/）。

**要点**: 后续所有通过 `document.cookie` 访问页面的请求，中间件都能从 cookie 中读取 token，认证正常通过。

## V3.1.0 侧边栏重构 + 业绩统计

| 分类 | 详情 |
|------|------|
| **版本** | V3.1.0 |
| **日期** | 2026-05-08 21:00→21:50 |
| **涉及文件** | `backend/routers/performance.py`, `frontend/templates/base.html` |
| **备份** | `projects/_backups/gym-web-system_v3.1.0_20260508_2150/` |

### Bug 1: 课程包卡片金额显示错误

**现象**: 业绩总览的课程包统计卡片显示的金额与售课卡片完全一样。

**根因**: `LessonPackage` 模型没有 `price` 字段，卡片从 `Sale` 关联查询价格，逻辑实际是售课而非课程包统计。

**修复**: 课程包统计改为展示**包数**和**课时数**，不再展示金额类数据（因课程包无独立价格字段）。

### Bug 2: 会籍卡"有效"统计忽略时间筛选

**现象**: 选择"本月"时间范围后，会籍卡统计显示"售出 6 张 / 有效 8 张"（有效数>售出数），时间筛选未作用于"有效"统计。

**根因**: 售出统计按时间筛选，但"有效"卡片统计忽略了时间条件。

**修复**: 有效统计也按相同时间段筛选。

### Bug 3: 导航下拉菜单回弹

**现象**: 鼠标移开导航下拉菜单立即消失，用户体验差。

**根因**: 顶栏下拉菜单使用 CSS hover 控制，鼠标稍微偏离就触发隐藏。

**修复**: 将顶部导航栏改为**左侧边栏**布局（固定 224px，深蓝背景），彻底消除下拉菜单回弹问题。

### Bug 4: LessonPackage 无 price 属性导致 500

**现象**: 访问业绩总览或课程包统计页面时返回 HTTP 500。

**根因**: `overview` 和 `packages` 统计接口访问 `LessonPackage.price`，但该模型没有此字段。

**修复**: 改为使用 `lesson_count` 和 `package_count` 作为展示指标。

## V3.1.2 商品零售购物车

| 分类 | 详情 |

| **备份** | `projects/_backups/gym-web-system_v3.1.2_20260508_2325/` |

### Bug: 批量下单时 generate_id 重复

**现象**: 批量下单（同一张订单中包含多个商品）时，所有记录的 ID 全部相同，导致唯一约束冲突。

**根因**: `generate_id()` 在单次 POST 的循环中被多次调用，但数据库中尚未提交插入记录，导致每次查询 Max ID 都返回相同的值。

**修复**: 循环前先计算 base ID，然后在循环中手动自增序号追加：
```python
base = db.query(func.max(ProductSale.id)).scalar() or 0
for i, item in enumerate(cart_items):
    new_id = base + i + 1
```

## V3.1.8-hotfix 期限卡签到 + 现金卡扣费

| 分类 | 详情 |
|------|------|
| **版本** | V3.1.8（热修复） |
| **日期** | 2026-05-09 18:18 |
| **备份** | `projects/_backups/gym-web-system_v3.1.8_期限卡修复_20260509_181545/` (94 files) |

### Bug 1: 期限卡签到选项不显示

**现象**: 选择期限卡会员后，核销方式选项中没有"期限卡签到"。

**根因 (核心)**:
- 数据库 `membership_card` 的 `card_type` 字段存储的是大类名称"期限卡"
- 但前端 JS 的条件判断使用细分子类型列表：
  ```javascript
  ['月卡','季卡','年卡','时卡'].includes(ctype)
  ```
- `ctype` 的实际值为"期限卡"，不在列表中 → 判断为 false → 选项未被创建

**受影响的文件（3 处）**:

| 文件 | 位置 | 改动 |
|------|------|------|
| `frontend/templates/checkin.html` | `onCardSelect()` | 期限卡判断增加 `ctype === '期限卡'` |
| `frontend/templates/index.html` | `has期限卡` 定义 | 增加 `c.card_type === '期限卡'` |
| `frontend/templates/index.html` | `showQuickOptionsFallback()` | 增加 `ctype === '期限卡'` |

### Bug 2: 现金卡扣费不能自定义金额

**现象**: 现金卡扣费预设 `value="30"`，用户无法手动输入扣费金额。

**修复（涉及 6 个文件/函数）**:

| 位置 | 改动前 | 改动后 |
|------|--------|--------|
| `checkin.html` → `onCardSelect()` | `<input value="30">` | `<input placeholder="输入金额">` |
| `checkin.html` → `onConsumeOptionChange()` | 现金卡不显示金额输入框 | 显示金额输入框 |
| `checkin.html` → `updateDeductPreview()` | — | 用输入框实际金额计算预览 |
| `checkin.html` → `submitCheckin()` | 从隐藏 `cardIdInput` 取 card_id | 从 `cardSelect` 取值 |
| `index.html` → 快速签到 | 各种硬编码 `value="30"` | 改为输入框，无默认值 |
| `checkin.py` → `create_checkin()` | 金额可选，不足时用默认值 30 | `consume_amount` 必传且 > 0，余额不足报错 |

### Bug 3: 浏览器缓存导致新版 JS 不生效

**现象**: 服务端已更新了 `checkin.html` 和 `index.html`，但浏览器一直加载旧版本。

**修复**: 在 `backend/app.py` 的 `/` 和 `/checkin` 路由中添加防缓存响应头：
```python
response.headers["Cache-Control"] = "no-cache, no-store, must-revalidate"
response.headers["Pragma"] = "no-cache"
response.headers["Expires"] = "0"
```

### Bug: 跟进员工下拉列表为空

**现象**: 会员编辑弹窗的"跟进员工"下拉菜单只有"—— 不指定 ——"一个选项，无法选择任何员工。

- ORM 模型 `Staff` 的 `status` 字段默认值为"在职"

2. **后端宽松过滤**: `staff_active` 端点增加 `Staff.status == ""` 条件：
   ```python
   filter(or_(Staff.status == "在职", Staff.status == ""))
   ```

## V3.2.1 — Tab 切换 hidden 类修复（2026-05-12 20:30）

**类型**: bugfix
**版本**: V3.2.1（热修复）
**备份**: `_backups/gym-web-system_v3.2.1_Tab切换hidden修复_20260512_2031/` (204 files)

### 问题 1: 产品管理 Tab "刷新后第1秒正常→变回旧版"
**原因**: 弹窗（productModal）内的 `#courseSelectContainer` 带有 `hx-get="/api/packages/courses-options" hx-trigger="load"`。页面加载时 HTMX 触发了这个请求，虽然目标不是 productTableArea，但 HTMX 加载过程触发了某些 CSS 状态重置。

**修复**: 移除 `hx-trigger="load"`，改为 DOMContentLoaded 回调中通过 JS 异步加载课程选择器。

### 问题 2: 已售课程包 Tab 切换后内容空白
**原因**: tailwindcss `hidden` 类（`display: none`）与 HTMX 的 DOM 操作存在 CSS 优先级冲突。`classList.toggle('hidden', ...)` 虽然正确执行，但 CSS 优先级规则导致 `hidden` 未生效。

**修复**: 放弃 tailwindcss `hidden` 类，改用**内联样式 `style.display`**：
- 默认隐藏: `<div id="tabMemberContent" style="display:none">`
- 切换: `document.getElementById(id).style.display = 'none' / ''`

### 影响范围
- `frontend/templates/packages.html` — `switchTab()` 函数重写、`tabMemberContent` 改用内联 style
- 后端无改动

### 教训
1. **内联 style 优先于 CSS class 控制显隐**：尤其在 HTMX 参与 DOM 操作的场景
2. **HTMX `hx-trigger="load"` 会异步加载**：即使元素在 hidden 弹窗中也会触发
3. **CSS 类冲突难以调试**：浏览器 Inspector 可见元素但不可见时，试试内联 style
