# 开发日志 V3.5.4 — 会员管理模块增强

**日期**: 2026-05-20
**类型**: feature + bugfix
**触发**: 用户反馈会员管理功能单调，要求重新设计

---

## 背景

会员管理模块原有功能单一：一个平铺的 10 列表格 + 关键词搜索 + 增删改弹窗。用户需要更丰富的功能：统计概览、多维度筛选、会员详情页（聚合所有关联数据）。

---

## 新增功能

### 1. 统计卡片

**端点**: `GET /api/members/stats` → HTML 片段

4 张卡片：总会员数 / 本月新增 / 本月到期 / 活跃率（有效会员占比）。

### 2. 多维筛选栏

**端点**: `GET /api/members/filter-options` → JSON

筛选维度：关键词搜索、等级、状态、来源、跟进员工、注册日期范围、最近签到日期范围。下拉选项从数据库去重获取。

### 3. 增强表格（可排序 + 分页）

**端点**: `GET /api/members/table`（改造）

- 新增"最近签到"和"来源"列（10 → 12 列）
- 表头点击排序（白名单校验防 SQL 注入）
- 分页控件：20 条/页，上一页/下一页/页码跳转
- 成员姓名变为可点击链接，跳转详情页
- 操作列增加"详情"按钮

### 4. 会员详情页

**页面路由**: `GET /members/{member_id}`

- 左侧：照片 + 完整资料（性别/手机/生日/身高/体重/体脂/来源/跟进/备注/手环）
- 中间：4 张摘要小卡片（累计签到/剩余课时/储值余额/到期日期+剩余天数）
- 右侧：快捷操作按钮（编辑/核销/充值/售课）
- 下方：5 个 Tab 切换区

### 5. Tab 懒加载（HTMX）

| Tab | 端点 | 说明 |
|-----|------|------|
| 会籍卡 | `GET /api/members/{id}/cards-html` | 类型徽章、剩余次数/金额、有效期、状态 |
| 进场记录 | `GET /api/members/{id}/checkins-html` | 最近 30 条，含 30 天统计 |
| 上课记录 | `GET /api/members/{id}/class-records-html` | 最近 30 条，教练、课程、消耗课时 |
| 体测记录 | `GET /api/members/{id}/body-measurements-html` | 全部记录 + 最新两次趋势（体重/体脂/BMI 变化箭头） |
| 消费记录 | `GET /api/members/{id}/purchases-html` | 充值+售课合并，按日期倒序 |

首 Tab（会籍卡）页面加载时自动获取，其余 Tab 首次点击时 HTMX 懒加载。

## BUG 修复清单

### BUG-5: 详情页缺少 `today` 上下文变量
| 项目 | 内容 |
|------|------|
| **文件** | `backend/app.py` |
| **端点** | `GET /members/{member_id}` |
| **现象** | 到期日期卡片"剩余天数"计算依赖 `today` 变量，模板渲染时报错 |
| **修复** | 在 `context` 中添加 `"today": date.today()` |

| 项目 | 内容 |
|------|------|
| **文件** | `backend/app.py` |
| **端点** | `GET /members/{member_id}` |
| **现象** | 访问不存在的会员 ID 时返回纯文本 "Internal Server Error" |
| **修复** | FastAPI import 中添加 `HTTPException` |

## 设计决策

1. **详情页用独立页面而非侧边面板**：关联数据量大（5 个 Tab 的表格），侧面板太拥挤
2. **HTML 片段端点而非 JSON + 前端渲染**：遵循项目 HTMX SSR 架构
3. **筛选用显式查询按钮而非自动提交**：多字段筛选若每次变更都发请求会产生过多请求
4. **排序白名单**：`SORT_WHITELIST` 字典映射列名到 ORM 字段，防止 SQL 注入
5. **分页替代硬编码 50 条限制**：详情页出现后用户会频繁来回浏览
