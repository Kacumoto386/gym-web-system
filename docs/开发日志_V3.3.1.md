# 开发日志 V3.3.1 — 预约管理支持日期范围查询

**日期**: 2026-05-13
**类型**: feature

---

## 需求

预约管理页面当前只支持按单日查询（一个 `<input type="date">`），改为支持按日期范围查询（起止日期）。

## 后端变更

### booking_list (JSON API)
- 新增 `date_from` / `date_to` Query 参数
- 保留旧 `date_str` 参数做兼容
- `date_from` → `Booking.booking_date >= date_from`
- `date_to` → `Booking.booking_date <= date_to`
- `date_str` 独立使用（等于查询）

### booking_table (HTML 片段)
- 同上新增 `date_from` / `date_to`
- 当 `date_from` 和 `date_to` 都为空时 fallback 到旧 `date_str`

## 前端变更

### booking.html
- 单个日期输入改为两个（`filterDateFrom` → `filterDateTo`），中间有"至"分隔
- 初始加载 hx-get 从 `?date_str={{ today }}` 改为 `?date_from={{ today }}&date_to={{ today }}`
- `refreshTable()` 改为读取两个日期并拼接 `date_from`/`date_to` 参数
- `todayFilter()` 同时设置起止日期为今天

## 文件变更清单

| 文件 | 操作 | 说明 |
|------|:----:|------|
| backend/routers/booking.py | 修改 | booking_list + booking_table 增加 date_from/date_to 参数 |
| frontend/templates/booking.html | 修改 | 单日期输入 → 双日期范围输入 |
| backend/app.py | 修改 | 版本号 3.3.1 |
| frontend/templates/base.html | 修改 | 版本号 v3.3.1 |

## 兼容性
- 旧版 `date_str` 参数保留，不破坏现有调用方
- 优先使用 `date_from`/`date_to`，只有两者都为空时才用 `date_str`
