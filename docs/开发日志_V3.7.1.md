# 开发日志 V3.7.1 — 商品零售进销存增强

**日期**: 2026-05-22  
**类型**: feature

---

## 变更内容

### 1. 进货入库管理

新增 `StockInbound` 模型，完整的进货入库 CRUD：

- `GET /api/products/inbounds/table` — 进货记录表格 HTML（按日期降序）
- `POST /api/products/inbounds` — 创建进货记录，自动增加 `Product.stock`
- `DELETE /api/products/inbounds/{inbound_id}` — 删除进货记录，自动回滚 `Product.stock`
- `GET /api/products/inbounds/form-options` — 商品下拉选项（入库弹窗用）
- 入库记录包含：入库编号、日期、商品、数量、单价、总成本、供应商、操作员

### 2. 安全库存与库存预警

- `Product` 模型新增 `min_stock`（安全库存）字段
- 商品表格中库存数字按颜色区分：红色加粗=缺货，橙色=低于安全库存，黑色=正常
- `GET /api/products/low-stock` — 低库存预警 HTML 表格
- 商品管理表单增加"安全库存"字段

### 3. 销售扣减库存 + 利润计算

- `ProductSale` 模型新增 `cost_price`（售出时成本价）和 `profit`（利润）字段
- 创建商品零售时自动扣减库存，记录成本价和利润
- 批量零售同样逐项扣减库存
- 删除/作废零售记录时自动归还库存
- 库存不足时返回 422 错误

### 4. 利润统计

- `GET /api/products/profit-summary` — 利润统计 HTML（汇总卡片 + 商品利润排行）
- 总销售额、总成本、总利润、利润率
- 按商品维度分组销量/销售额/利润/利润率

### 5. 盘点调整

- `PUT /api/products/{product_id}/adjust-stock` — 盘点调整库存
- 接收新库存数和调整原因，记录操作日志

### 6. 商品编辑功能

- `PUT /api/products/{product_id}` — 编辑商品所有字段
- 商品管理表格增加"编辑"和"盘点"操作按钮

### 7. 仪表盘 API 迁移

- 首页仪表盘统计 API 从 `app.py` 迁移至 `backend/routers/dashboard.py`
- `dashboard_stats`、`today_bookings_dashboard`、`today_checkins_dashboard` 统一归入 dashboard 路由

### 8. 操作日志中间件修复

- `str.lstrip()` → `path[len(prefix):]` 切片修复（lstrip 会逐字符剥离导致 bug）
- 增加 `_KNOWN_ACTION_SEGMENTS` 防止非 ID 路径段被误当作 resource_id
- 扩充 `_RESOURCE_CN_MAP` 覆盖更多资源中文名
