# 开发日志 V3.8.2 — 数据导入模块

**日期**: 2026-05-25  
**类型**: feat

---

## 变更内容

### 1. 新增数据导入模块（通用框架）

**改动**：
- 新增 `backend/routers/import_data.py` — 完整的数据导入框架
- 支持 4 种导入类型：会员、员工、会籍卡、已售私教课
- 统一流程：模板下载 → 上传解析 → 预览确认 → 后台执行 → 进度跟踪

### 2. 模板下载（`GET /api/import/template/{type}`）

- 根据类型动态生成标准 Excel 模板（.xlsx）
- 蓝色加粗表头行 + 灰色示例行（第二行展示格式说明）
- 必填字段以 `*` 标记（会员：姓名/手机号；员工：姓名/手机号；会籍卡：会员手机号/姓名；私教课：会员手机号/姓名）

### 3. 上传解析（`POST /api/import/upload`）

- 中文表头自动映射（与模板一致）
- 必填列缺失时返回明确错误提示
- 日期字段自动解析（support both datetime objects and ISO strings）
- Decimal 字段类型安全转换（空值 → Decimal(0)）
- 空行自动跳过

### 4. 后台执行（`POST /api/import/{task_id}/execute` + threading.Thread）

- 使用 `threading.Thread(daemon=True)` 确保后台可靠执行（避免 BackgroundTasks 在同步端点中失效）
- `key_field` 配置驱动 UPSERT vs INSERT 模式：
  - 员工（phone）→ UPSERT
  - 会员/会籍卡/私教课 → 始终 INSERT
- PROTECTED_FIELDS 防止系统字段被覆盖（ID、余额、课时、创建时间等）

### 5. 进度跟踪（`GET /api/import/{task_id}/progress`）

- 实时返回：总行数/成功/失败/新建/更新/跳过计数
- 错误详情逐行展示
- 前端 1500ms 轮询进度
- 完成/失败自动停止轮询

### 6. 导入历史（`GET /api/import/history/table` + `GET /api/import/history`）

- HTML 表格（HTMX 加载） + JSON 列表双端点
- 状态展示：待确认 → 导入中 → 已完成 / 失败
- 待确认任务可点击"执行导入"

### 7. 前端新增导入页面（`frontend/templates/import.html`）

- 4 类型 Tab 切换（会员/员工/会籍卡/私教课）
- 3 步操作流：下载模板 → 上传文件 → 确认导入
- 实时进度条 + 统计数字（总行数/成功/失败/新建/更新）
- 错误详情区域 + 导入历史表格
